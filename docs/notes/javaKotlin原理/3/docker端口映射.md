如果docker有两个程序容器映射到同一个端口，外部访问这个端口时访问的是哪个程序？

**哪个都访问不到**——Docker 会**直接报错**，**拒绝启动第二个容器**。

------------------------------------------------
1. 底层机制
- Docker 宿主机上的**端口资源（IPv4:PORT）是唯一的**。  
- 当第二个容器执行 `docker run -p 80:8080 ...` 时，Daemon 会尝试向内核申请 **“绑定 0.0.0.0:80”**；  
  若已被第一个容器占用，内核返回 `EADDRINUSE`，Daemon 立即抛出：
  ```
  Error response from daemon: driver failed programming external connectivity on endpoint xxx:
  bind: address already in use
  ```

------------------------------------------------
2. 实验验证
```bash
# 第一个容器
docker run -d --name web1 -p 80:80 nginx:alpine

# 第二个容器
docker run -d --name web2 -p 80:80 nginx:alpine
```
输出：
```
Error ... bind: address already in use
```
**web2 无法创建**，自然**不存在“访问谁”的问题**。

------------------------------------------------
3. 想让两个容器**同时对外服务**的正规做法
| 方案                         | 说明                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| **映射到不同宿主机端口**     | `-p 81:80`、`-p 82:80`                                       |
| **使用反向代理**             | Nginx/HaProxy/Caddy 统一监听 80，按域名/路径转发到容器内部 IP:Port |
| **Docker Swarm/K8s Ingress** | 由负载均衡器根据规则分发流量                                 |

------------------------------------------------
一句话结论
> **同一宿主机端口只能被一个容器占用；**  
> **第二个容器会启动失败并提示 `address already in use`；**  
> **想并行服务 → 换端口或上层反向代理。**
