count(DISTINCT UID, DATE_FORMAT(submit_time, '%Y%m%d')这个在算什么？

这是在算：

> **「每个用户每天最多只被计数一次」** 的 **去重组合键**。

---

### 拆解字段

| 部分                                 | 含义               |
| ------------------------------------ | ------------------ |
| `UID`                                | 用户唯一标识       |
| `DATE_FORMAT(submit_time, '%Y%m%d')` | 提交日期（年月日） |

→ `COUNT(DISTINCT UID, 日期)` = **(用户, 日期) 组合去重后计数**  
→ 结果 = **「有多少个『用户-日期』对」**，即「每天每用户最多算 1 次」。

---

### 现场例子

| UID  | submit_time      |
| ---- | ---------------- |
| A    | 2023-04-01 08:00 |
| A    | 2023-04-01 20:00 |
| B    | 2023-04-01 10:00 |
| A    | 2023-04-02 08:00 |

→ `COUNT(DISTINCT UID, DATE_FORMAT(...))` = **3**  
（对：A-20230401, B-20230401, A-20230402）

---

### 一句话背下来

> **「COUNT(DISTINCT 用户, 日期) = 每天每用户最多算 1 次，结果 = 用户-日期对数。」**



SELECT DATE_FORMAT(submit_time, '%Y%m') MONTH,
                                        round(count(DISTINCT UID, DATE_FORMAT(submit_time, '%Y%m%d')) / count(DISTINCT UID), 2) avg_active_days,
                                        COUNT(DISTINCT UID) mau
FROM exam_record
WHERE YEAR (submit_time) = 2021
GROUP BY MONTH这个month是什么？表里没有这个字段

`MONTH` 是 **你给 SELECT 列起的别名**，不是表里的原始字段。

写法拆解：

```sql
SELECT DATE_FORMAT(submit_time, '%Y%m')  AS MONTH,
       ...
FROM exam_record
WHERE YEAR(submit_time) = 2021
GROUP BY MONTH;   -- 这里直接用了上面刚起的别名
```

- 表里没有 `month` 列，只有 `submit_time`（datetime）
- `DATE_FORMAT(submit_time, '%Y%m')` 把日期截成年月字符串
- 你在 SELECT 里给它起了别名 `MONTH`
- `GROUP BY MONTH` 就是按这个「年月字符串」分组

---

### 一句话背下来

> **「MONTH 是你刚起的别名，不是表里字段；GROUP BY 可以直接用 SELECT 里的别名。」**
