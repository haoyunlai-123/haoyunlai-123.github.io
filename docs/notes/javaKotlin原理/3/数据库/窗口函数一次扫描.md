WITH a AS (
SELECT * FROM examination_info INNER JOIN exam_record USING(exam_id) 
WHERE tag = 'SQL' AND submit_time IS NOT NULL AND score IS NOT NULL
)
SELECT MIN(score) FROM a WHERE score >= (SELECT AVG(score) FROM a);这句sql对吗？

**语法完全正确**，逻辑也成立，但可以再「瘦一点」：

1. **WHERE 里两次扫 a**，MySQL 会执行两遍子查询；可用 **CTE + HAVING** 或 **派生表+变量** 避免重扫。
2. **MIN(score) WHERE score >= AVG** 其实就是「高于平均的最小值」，**用窗口函数一行就能拿**。

---

### 更瘦写法（窗口函数，只扫一次）

```sql
WITH a AS (
    SELECT score, AVG(score) OVER () AS avg_score
    FROM examination_info
    JOIN exam_record USING (exam_id)
    WHERE tag = 'SQL'
      AND submit_time IS NOT NULL
      AND score IS NOT NULL
)
SELECT MIN(score) AS min_above_avg
FROM a
WHERE score >= avg_score;
```

---

### 一句话背下来

> **「你的写法对，但重扫 a 两次；用窗口函数 AVG OVER () 一次扫完，性能更干净。」**
