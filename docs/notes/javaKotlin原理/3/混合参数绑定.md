@RequestMapping("/simplePojo")
    public String simplePojo(User user) {
        System.out.println(user);

非常好的问题 👏——你现在问到的是 **Spring MVC 参数绑定机制中一个非常有代表性的“坑”**，
它恰好体现了 Spring 是如何把 **URL 参数 + 表单参数** 混合绑定到一个 Java Bean（Pojo）上的。

我们来完整拆解你这个例子的过程：

---

# 🌐 一、你的例子复现

```java
@PostMapping("/simplePojo")
public String simplePojo(User user) {
    System.out.println(user);
    return "ok";
}
```

请求：

```
POST http://localhost:8080/simplePojo?name=xiaoyage&age=12
Content-Type: application/x-www-form-urlencoded

name=lanmao&age=18
```

输出结果：

```
User{name='xiaoyage,lanmao', age=12}
```

👉 name 字段成了两个值拼接：`"xiaoyage,lanmao"`
👉 age 字段取的是 URL 中的 `12`

看似“怪异”，其实是完全符合 Spring MVC 的绑定机制。

---

# 🧩 二、Spring MVC 的参数绑定机制原理

当你的方法参数是一个 **Pojo（例如 User）** 时，Spring MVC 会这样处理：

1. **由 `ServletModelAttributeMethodProcessor` 处理 `User` 参数；**
2. 它内部调用 `WebDataBinder`；
3. `WebDataBinder` 从 `request.getParameterMap()` 获取所有参数；
4. 使用 Java Bean 的属性名反射调用 `setName()`、`setAge()`；
5. 如果同名参数出现多个值（例如来自 URL + Body），它们都会进入同一个属性绑定流程。

---

# ⚙️ 三、关键细节：Servlet 参数 Map 是 “多值合并” 的

### Servlet 的请求参数结构：

`HttpServletRequest.getParameterMap()` 返回：

```java
Map<String, String[]> parameters;
```

在你的例子中，它实际包含：

```java
{
  "name": ["xiaoyage", "lanmao"],
  "age": ["12", "18"]
}
```

也就是说：

* URL 上的参数、body 中的参数都被 **合并成数组**；
* `getParameter("name")` 通常会取数组第一个或拼接后的字符串；
* `getParameterValues("name")` 则能拿到全部值。

---

# 🧠 四、Spring 的绑定策略（Pojo 版）

当是 **表单参数绑定到 Pojo** 时，Spring 的 `DataBinder` 使用 `BeanWrapperImpl` 去绑定字段：

* 如果字段类型是 **数组 / List** → 它会把所有值都放进去；
* 如果字段类型是 **单个 String** → Spring 默认会把多个值 **拼接成一个字符串，用逗号分隔**；
* 如果字段类型是 **int/Integer** → 取第一个可转换的值。

所以在你的例子中：

| 字段     | 值来源                      | Spring 处理结果             |
| ------ | ------------------------ | ----------------------- |
| `name` | `["xiaoyage", "lanmao"]` | 合并为 `"xiaoyage,lanmao"` |
| `age`  | `["12", "18"]`           | 取第一个 `"12"` 转成 int      |

---

# 📘 五、验证这个行为

你可以打印：

```java
@RequestMapping("/simplePojo")
public String simplePojo(HttpServletRequest req) {
    System.out.println(Arrays.toString(req.getParameterValues("name")));
    System.out.println(Arrays.toString(req.getParameterValues("age")));
    return "ok";
}
```

输出将会是：

```
[name=xiaoyage, lanmao]
[age=12, 18]
```

说明 Spring 并没有覆盖，而是“收集全部”，再决定怎么处理。

---

# 🧩 六、那为什么 `age` 没合并？

因为 `age` 在 Java Bean 里定义是整数类型：

```java
private int age;
```

Spring 尝试把字符串数组 `["12", "18"]` 转为 int，只能取第一个元素成功。
因此结果为 `age=12`。

而 `name` 是字符串字段：

```java
private String name;
```

所以多个值会用 `","` 拼接成一个字符串。

---

# ✅ 七、总结结论表

| 情况                   | 说明                     | 结果            |
| -------------------- | ---------------------- | ------------- |
| URL 与 Body 都含同名参数    | Servlet 合并为 `String[]` | Spring 会看到多个值 |
| 字段类型为 `String`       | 多值拼接为 `"val1,val2"`    | ✅             |
| 字段类型为 `int/Integer`  | 取第一个可转换值               | ✅             |
| 字段类型为 `List<String>` | 全部加入 List              | ✅             |

---

# 💡 八、如何避免混合绑定的问题

有几种方式：

### ✅ 方法 1：只保留一种参数来源

只用 URL 或只用 Body，不混用相同名称。

### ✅ 方法 2：显式区分参数来源

```java
public String simplePojo(
    @RequestParam String name,  // URL 参数
    @RequestBody User user      // Body 参数
)
```

这样 Spring 不会合并两个来源。

### ✅ 方法 3：使用 `@InitBinder` 自定义参数绑定逻辑

如果确实需要特殊逻辑，可在 Controller 内自定义参数绑定器。

---

# 🚀 九、一句话总结

> 在 Spring MVC 中，URL 参数与表单参数会被 Servlet 合并到一个多值 Map 中；
> 当绑定到 `Pojo` 时：
>
> * **String 字段：** 多值会拼接成 `"val1,val2"`；
> * **数值字段：** 取第一个；
> * **集合字段：** 全部保存。
