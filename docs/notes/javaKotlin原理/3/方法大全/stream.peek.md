stream.peek()是什么方法？

`Stream.peek()` 是 Java 8 Stream API 提供的**中间操作**，一句话作用：

> **“对流中每个元素执行一次副作用操作（如打印、日志、修改状态），然后原样返回该元素，不改变流内容。”**

------------------------------------------------
典型用法（调试/日志）
```java
List<String> list = Arrays.asList("a", "b", "c");
list.stream()
    .peek(System.out::println)   // 副作用：打印
    .map(String::toUpperCase)
    .collect(Collectors.toList());
```
输出：
```
a
b
c
[A, B, C]
```

------------------------------------------------
与 `map` 的区别
| 方法      | 返回           | 用途                       |
| --------- | -------------- | -------------------------- |
| `map(f)`  | **转换后**元素 | 修改/转换流内容            |
| `peek(f)` | **原样**元素   | 副作用（调试、日志、计数） |

------------------------------------------------
一句话背走
> **peek = 副作用观察口，不改变流，常用于调试/日志；**  
> **不要用它做业务转换，否则代码可读性下降。**
