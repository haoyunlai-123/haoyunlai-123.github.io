```java
public void handleDelayTask(){
        while (begin) {
            try {
                // 1.获取到期的延迟任务
                DelayTask<RecordTaskData> task = queue.take();
                RecordTaskData data = task.getData();

```

`@Transactional` 想生效，必须同时满足两个前提：

1. **Spring 代理对象** 来调用目标方法；  
2. 调用链路上**没有捕获异常并吞掉**（否则 Spring 感知不到异常就不会回滚）。

你的代码里两条都违反了：

------------------------------------------------
问题 1：自己人调自己人 —— 绕过了代理
------------------------------------------------
```java
// 假设类名叫 DelayTaskService
@Transactional
public void handleDelayTask() { … }   // ① 标注了事务
```
而**真正启动 while 循环的线程**是：
```java
new Thread(() -> delayTaskService.handleDelayTask()).start();   // ② 直接拿到原始对象
```
或者
```java
executor.execute(() -> delayTaskService.handleDelayTask());     // ② 同样直接引用
```
Lambda 里拿到的 `delayTaskService` 是**Spring 容器里的原始 Bean**，不是 CGLIB 代理，因此 `@Transactional` 切面不生效。

------------------------------------------------
问题 2：异常被 catch 并记录日志 —— Spring 捕获不到
------------------------------------------------
```java
try {
    ...
} catch (Exception e) {
    log.error("处理延迟任务发生异常", e);   // ③ 异常被吞掉
}
```
即使代理生效，Spring 也只能在**抛出异常时**回滚；现在异常被抓住，事务便认为“一切正常”而提交。

------------------------------------------------
解决思路
------------------------------------------------
1. **把事务边界拆到小方法**  
   把“真正需要事务的代码”抽出去，让 Spring 代理对象调用：

   ```java
   @Component
   public class DelayTaskService {
       private final ExecutorService executor = Executors.newSingleThreadExecutor();
   
       @PostConstruct
       public void start() {
           // 启动后台线程，注意这里用代理对象
           executor.execute(() -> handleDelayTask());
       }
   
       public void handleDelayTask() {
           while (begin) {
               try {
                   DelayTask<RecordTaskData> task = queue.take();
                   // 用代理对象调用含事务的方法
                   SpringContext.getBean(DelayTaskService.class).process(task);
               } catch (InterruptedException ie) {
                   Thread.currentThread().interrupt();
                   break;
               } catch (Exception e) {
                   log.error("处理延迟任务发生异常", e);
                   // 不吞异常，也可以直接抛出去让线程工厂处理
               }
           }
       }
   
       @Transactional(rollbackFor = Exception.class)
       public void process(DelayTask<RecordTaskData> task) {
           RecordTaskData data = task.getData();
           LearningRecord record = readRecordCache(data.getLessonId(), data.getSectionId());
           if (record == null) return;
           if (!Objects.equals(data.getMoment(), record.getMoment())) return;
   
           record.setFinished(null);
           recordMapper.updateById(record);
   
           LearningLesson lesson = new LearningLesson();
           lesson.setId(data.getLessonId());
           lesson.setLatestSectionId(data.getSectionId());
           lesson.setLatestLearnTime(LocalDateTime.now());
           lessonService.updateById(lesson);
       }
   }
   ```
   关键点：
   - `SpringContext.getBean(DelayTaskService.class)` 拿到的是 **CGLIB 代理**，事务切面才能织入。  
   - 异常在 `process` 方法里抛，Spring 捕获后回滚；外层 `handleDelayTask` 只负责线程循环和日志，不再吞异常。

2. **或者把事务放到 Service 层**  
   把第 4 步封装成一个 `LearningProgressService.recordLearning(...)` 方法并加 `@Transactional`，然后注入调用即可，原理同上。

3. **若使用声明式事务，一定注意**  
   - 同类内自调（`this.xxx()`）无效 → 必须拆方法或自己拿代理。  
   - 异常被 catch 后需要 **手动** `TransactionAspectSupport.currentTransactionStatus().setRollbackOnly()` 才能回滚。

------------------------------------------------
一句话总结
------------------------------------------------
**`@Transactional` 只对“Spring 代理对象”的公开方法生效；**  
你在 while 线程里直接调用同类方法 + 把异常 catch 掉，事务切面既进不来又感知不到异常，自然就不会回滚。  
把事务代码拆到小方法、用代理对象调用、别再吞异常，就能生效。
