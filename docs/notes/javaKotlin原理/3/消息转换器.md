```java
/**
     * Spring Mvc框架扩展消息转化器
     * 对后端返回给前端的数据进行统一处理
     * @param converters
     */
    protected void extendMessageConverters(List<HttpMessageConverter<?>> converters) {
        log.info("扩展消息转换器....");
        //创建一个消息转换器对象
        MappingJackson2HttpMessageConverter conveter = new MappingJackson2HttpMessageConverter();
        //为消息转换器对象创建一个转换器对象，可以将Java对象序列化为json数据
        conveter.setObjectMapper(new JacksonObjectMapper());
        //将自己的消息转换器加入Spring集合中
        converters.add(0,conveter);
    }这段代码是干啥的？
```
非常好 👏，你现在问到的这一段代码 ——
`extendMessageConverters()` —— 正是 **Spring MVC 底层的 HTTP 消息转换机制（HttpMessageConverter）** 的一个扩展点。
这段代码的作用是：

> 🔹 **统一定制 Spring MVC 把 Java 对象转成 JSON（或者反过来解析 JSON）的逻辑。**
>
> 换句话说，它控制了后端 Controller 返回对象 → 转成 JSON → 发给前端 的过程。

---

## 🧩 一、背景知识：Spring MVC 如何把 Java 对象变成 JSON？

比如我们写：

```java
@GetMapping("/user")
public User getUser() {
    return new User("Tom", 18);
}
```

浏览器收到：

```json
{"name": "Tom", "age": 18}
```

这一过程是谁做的？
👉 就是 Spring MVC 内部的 `HttpMessageConverter` 完成的。

Spring MVC 会根据请求和响应类型自动选择合适的 Converter：

| Converter                             | 功能      |
| ------------------------------------- | ------- |
| `StringHttpMessageConverter`          | 处理字符串   |
| `FormHttpMessageConverter`            | 处理表单    |
| `MappingJackson2HttpMessageConverter` | 处理 JSON |
| `ByteArrayHttpMessageConverter`       | 处理二进制   |

所以当返回对象时，Spring 会找到 JSON 转换器（Jackson），自动执行：

```java
objectMapper.writeValueAsString(returnValue)
```

## ⚙️ 二、这段代码的作用

```java
protected void extendMessageConverters(List<HttpMessageConverter<?>> converters) {
    log.info("扩展消息转换器....");
    // 创建一个 JSON 消息转换器
    MappingJackson2HttpMessageConverter converter = new MappingJackson2HttpMessageConverter();
    // 指定用自己的 ObjectMapper（定制 JSON 序列化规则）
    converter.setObjectMapper(new JacksonObjectMapper());
    // 把自定义的放到列表最前面（优先使用）
    converters.add(0, converter);
}
```

逐行解释👇：

1. **Spring 在启动时会调用 `WebMvcConfigurer.extendMessageConverters()`**

   * 这是一个“框架扩展钩子方法”；
   * 允许开发者在默认 Converter 集合基础上添加或替换自定义的 Converter。

2. **创建一个 `MappingJackson2HttpMessageConverter`**

   * 这个类是 Jackson 官方提供的 JSON 序列化转换器；
   * 能将 Java 对象 ↔ JSON 字符串 自动转换。

3. **替换内部的 `ObjectMapper`**

   ```java
   converter.setObjectMapper(new JacksonObjectMapper());
   ```

   * 默认的 `ObjectMapper` 配置很基础；
   * 开发中通常需要定制，比如：

     * 忽略空字段；
     * 格式化时间；
     * 允许单引号；
     * 统一属性命名规则；
   * 所以你这里用自己的 `JacksonObjectMapper`（一般是一个自定义配置类）。

4. **插入到 converters 集合最前面**

   ```java
   converters.add(0, converter);
   ```

   * Spring MVC 在序列化响应时，会**从前往后遍历** converters；
   * 把你的放在第一个，确保优先使用你的规则。

## 🧠 三、运行时效果

执行流程大致是：

```
Controller 返回对象 → HandlerAdapter → ResponseBodyAdvice → HttpMessageConverter → ObjectMapper → JSON → ResponseBody
```

你注册的转换器会：

* 拦截返回值；
* 用自定义 `ObjectMapper` 序列化；
* 输出符合你要求的 JSON。

## 🧩 四、实际用途举例

你可以在 `JacksonObjectMapper` 中统一配置，比如：

```java
public class JacksonObjectMapper extends ObjectMapper {
    public JacksonObjectMapper() {
        super();
        // 忽略空字段
        this.setSerializationInclusion(JsonInclude.Include.NON_NULL);
        // 格式化时间
        this.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);
        this.setDateFormat(new SimpleDateFormat("yyyy-MM-dd HH:mm:ss"));
        // 支持 Java 8 时间类
        this.registerModule(new JavaTimeModule());
    }
}
```

这样所有返回的 JSON 格式都会自动遵守这些规则：

```json
{
  "id": 1,
  "name": "Tom",
  "createTime": "2025-10-23 10:15:00"
}
```

## ✅ 五、总结一句话

| 行为                                        | 作用                   |
| ----------------------------------------- | -------------------- |
| 方法名 `extendMessageConverters()`           | 是 Spring MVC 的扩展点    |
| 关键类 `MappingJackson2HttpMessageConverter` | 把 Java 对象 ↔ JSON     |
| 自定义 `JacksonObjectMapper`                 | 控制序列化行为（时间、空值、字段命名等） |
| `add(0, converter)`                       | 让自定义转换器优先级最高         |

>
> `extendMessageConverters()` 就是你插进来告诉它：
> “以后都按我这套规则输出 JSON！”
