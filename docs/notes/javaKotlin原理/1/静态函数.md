Kotlin 的类中**无法直接定义静态函数**（类似 Java 的 `static` 方法），但可通过以下方式实现等效功能：

---

### **一、替代方案与实现方式**
#### 1. **伴生对象（`companion object`）**  
在类内部声明伴生对象，将“静态”函数定义于其中，通过类名直接调用：  
```kotlin
class MyClass {
    companion object {
        fun staticMethod() { 
            println("This is a static-like method.")
        }
    }
}
// 调用方式
MyClass.staticMethod()  // 无需实例化
```

#### 2. **顶层函数（Top-level Functions）**  
在类外部直接定义函数（不属于任何类），通过包名或导入直接访问：  
```kotlin
// File: Utils.kt
fun globalStaticMethod() { 
    println("This is a top-level function.")
}
// 调用方式
globalStaticMethod()
```

#### 3. **`object` 单例**  
通过单例对象封装方法，实现全局唯一实例的“静态”调用：  
```kotlin
object MySingleton {
    fun staticMethod() { 
        println("This is a singleton method.")
    }
}
// 调用方式
MySingleton.staticMethod()
```

#### 4. **`@JvmStatic` 注解（Java 互操作）**  
在伴生对象中使用 `@JvmStatic` 注解，生成真正的 Java 静态方法：  
```kotlin
class MyClass {
    companion object {
        @JvmStatic
        fun jvmStaticMethod() {
            println("This is a JVM static method.")
        }
    }
}
// Java 中调用：MyClass.jvmStaticMethod();
```

---

### **二、方案对比与选择建议**
| **方案**      | **优点**                     | **缺点**                         | **适用场景**             |
| ------------- | ---------------------------- | -------------------------------- | ------------------------ |
| 伴生对象      | 语法简洁、可直接通过类名调用 | 实际为伴生对象实例方法           | 类相关的工具方法         |
| 顶层函数      | 无类依赖、全局可用           | 无法访问类的私有成员             | 通用工具函数             |
| `object` 单例 | 支持状态管理、线程安全       | 可能引入不必要的单例开销         | 全局共享逻辑或配置管理   |
| `@JvmStatic`  | 兼容 Java 静态方法调用       | 需额外注解、仅优化 Java 互操作性 | 需与 Java 代码交互的场景 |

---

### **三、核心结论**
1. Kotlin 设计上**弱化了静态方法的概念**，推荐使用伴生对象或顶层函数替代。  
2. 若需与 Java 互操作，优先选择 `companion object` + `@JvmStatic` 组合。  
3. 对于全局工具方法，顶层函数是最简洁的实现方式。
