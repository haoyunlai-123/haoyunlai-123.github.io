在 Kotlin 中，**传统的 `for` 循环的循环变量是只读的**，无法直接修改。但可以通过以下方式间接调整迭代逻辑：

---

### **1. 改用 `while` 循环（推荐）**
手动控制循环变量，自由修改其值：
```kotlin
var i = 0
while (i < 10) {
    println(i)
    if (i == 5) {
        i += 2  // 直接修改循环变量
        continue
    }
    i++
}
```
**输出**：`0 1 2 3 4 5 7 8 9`  
**用途**：处理需要动态调整循环变量的场景（如跳过特定索引）。

---

### **2. 结合高阶函数跳过元素**
通过集合操作间接控制迭代范围：
```kotlin
(0 until 10).forEach { index ->
    if (index in 5..6) return@forEach  // 跳过 5 和 6
    println(index)
}
```
**输出**：`0 1 2 3 4 7 8 9`  
**优势**：保持函数式风格，无需手动管理变量。

---

### **3. 使用区间迭代 + 标签控制**
通过 `step` 和标签限制行为：
```kotlin
loop@ for (i in 0 until 10) {
    when (i) {
        3 -> continue@loop   // 跳过 3
        7 -> break@loop      // 结束循环
        else -> println(i)
    }
}
```
**输出**：`0 1 2 4 5 6`  
**适用场景**：简单跳过或提前终止循环。

---

### **对比总结**
| **方法**     | **适用场景**                 | **可变性** | **代码简洁性** |
| ------------ | ---------------------------- | ---------- | -------------- |
| `while` 循环 | 需动态调整变量（如索引跳跃） | 完全可控   | 中             |
| 高阶函数     | 固定跳过规则                 | 不可变     | 高             |
| 标签控制     | 简单跳过或终止               | 有限       | 中             |

---

### **最佳实践**
- **优先选择 `while` 循环**：当需要精细控制循环变量时（如遍历链表、动态跳过索引）。
- **函数式操作优先**：若无需修改变量，使用 `forEach` 或集合操作提升代码可读性。
- **避免复杂逻辑嵌套**：必要时使用 `break` 或 `continue` 简化流程。

通过上述方法，可以在 Kotlin 中优雅地实现类似于修改循环变量的逻辑。
