在 Kotlin 中，`Array<T>` 的构造函数需要 **两个参数**：数组大小和一个用于初始化每个元素的 `lambda` 表达式。你的代码 `Array<IntArray>(n)` 只传入了大小 `n`，但缺少了初始化元素的逻辑，因此会报错。

---

### **错误原因**
Kotlin 的 `Array` 构造函数定义如下：
```kotlin
class Array<T> constructor(
    size: Int,
    init: (Int) -> T  // 必须提供初始化元素的 lambda
)
```
- 必须显式指定每个元素的初始化方式，无法直接通过 `Array<IntArray>(n)` 隐式创建未初始化的数组。
- 若省略初始化逻辑（如 `Array(n) { ... }`），编译器会报错：
  ```
  No value passed for parameter 'init' (未提供 'init' 参数)
  ```

---

### **修复方法**
根据需求，为数组的每个元素赋予初始值。以下是常见场景的修复方案：

#### 1. **初始化二维数组**
若需创建二维数组（`Array<IntArray>`），例如 `n x m` 的矩阵：
```kotlin
val rows = n      // 行数
val cols = 5      // 假设每列长度为 5
val arr = Array(rows) { IntArray(cols) }  // 初始化每行为一个长度为 cols 的 IntArray

// 等价于：
// arr[i] = IntArray(cols) 对每个 i
```

#### 2. **初始化空二维数组**
若需每行暂时为空数组（后续动态填充）：
```kotlin
val arr = Array(n) { IntArray(0) }  // 初始化为空 IntArray
```

#### 3. **自定义初始化逻辑**
通过 `lambda` 自定义每个元素的生成规则（例如赋默认值）：
```kotlin
val arr = Array(n) { index -> 
    IntArray(size = 3) { 0 }  // 每个元素是长度为 3 且全为 0 的 IntArray
}
```

---

### **关键区别：`Array<IntArray>` vs `IntArray`**
- **`Array<IntArray>`**：对象数组，每个元素是 `IntArray` 类型（相当于二维数组）。
- **`IntArray`**：基本类型数组（对应 Java 的 `int[]`），直接存储 `Int` 值。

---

### **错误示例与修正对比**
| **错误代码**                   | **修正后的代码**                     | **说明**                    |
| ------------------------------ | ------------------------------------ | --------------------------- |
| `val arr = Array<IntArray>(n)` | `val arr = Array(n) { IntArray(0) }` | 补充初始化 lambda           |
| `Array<Int>(5)`                | `IntArray(5)` 或 `Array(5) { 0 }`    | 基础类型数组需用 `IntArray` |

---

### **其他替代方案**
1. **使用 `arrayOfNulls` 创建可空数组**（仅当元素可为 `null` 时适用）：
   ```kotlin
   val arr = arrayOfNulls<IntArray>(n)  // 初始化为 n 个 null
   ```

2. **动态填充数组**：
   ```kotlin
   val arr = Array(n) { mutableListOf<Int>() }  // 每行为可变的 List<Int>
   ```

---

### **总结**
- **强制初始化**：Kotlin 严格要求数组元素必须显式初始化，避免未定义行为。
- **语法核心**：`Array(size) { initLambda }` 是唯一合法的构造函数调用形式。
- 若目标是创建一维数组，应直接使用 `IntArray(n)`：
  ```kotlin
  val arr = IntArray(n)  // 一维数组，元素默认初始化为 0
  ```
