你的代码报错是因为 **标签（Label）的作用域使用错误**。在 Kotlin 中，`it` 是 lambda 表达式的默认参数名，而 `outer@` 和 `inner@` 是标签，用于控制 `return` 或 `break`/`continue` 的跳转目标，**不能直接用于引用外层 lambda 的 `it`**。

---

### **错误原因**
1. **`it` 的默认作用域**  
   - 在 `forEach inner@ { ... }` 里，`it` 默认指代当前 `forEach` 的迭代元素（即 `prices` 的索引），而不是外层的 `ArrayDeque`。
2. **`@outer` 标签的误用**  
   - `it@outer` 的语法是错误的，标签不能直接用于访问外层 lambda 的 `it`。

---

### **修正代码**
如果需要在 `forEach` 内部访问外层的 `ArrayDeque`，可以通过以下两种方式：

#### **方法 1：显式命名外层 lambda 参数**
```kotlin
ArrayDeque<Int>().also { deque ->  // 显式命名参数为 deque
    (prices.size - 1 downTo 0).forEach { index ->
        while (deque.isNotEmpty()) {  // 直接使用 deque
            // ...
        }
    }
}
```

#### **方法 2：通过自定义标签控制流程（不推荐）**
```kotlin
ArrayDeque<Int>().also outer@ { deque ->
    (prices.size - 1 downTo 0).forEach inner@{ index ->
        while (deque.isNotEmpty()) {  // 直接使用 deque
            // 如需跳转，可用标签（例如 break@outer）
        }
    }
}
```

---

### **关键点**
- **`it` 是隐式参数名**，在嵌套 lambda 中会被内层覆盖。  
- **标签的作用是控制流程跳转**（如 `return@outer`），而非访问变量。  
- 最佳实践是显式命名 lambda 参数（如 `deque`），避免混淆。
