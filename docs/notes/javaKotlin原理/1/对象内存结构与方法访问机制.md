在 Java 内存模型中，**对象存储在堆区时并不直接保存方法的具体地址**，而是通过隐式指针间接访问方法区中的方法信息。具体机制如下：

---

### 一、对象内存结构与访问机制
1. **堆区对象组成**  
   每个对象在堆中存储以下核心数据：
   - **实例变量**：非静态成员变量的值。
   - **对象头**：包含锁相关信息和指向方法区中对应的类元数据（即 `Klass` 结构）的指针。（即**类型指针**）

   ![对象内存结构示意图](C:\Users\Administrator\Documents\Learn\kotlin和java\对象在堆区及方法区的分布.png)

2. **方法区的作用**  
   - 存储类的元数据，包括：
     - **方法字节码**：类的所有方法的具体实现代码。
     - **方法表（vtable）**：记录方法的实际入口地址（用于动态绑定和多态）。
   - 类元数据在类加载时被加载到方法区，而非堆中。

---

### 二、方法调用流程
1. **定位方法入口**  
   当通过对象调用方法时，JVM 会：
   - **Step 1**: 通过对象头中的类型指针，找到方法区中对应的类元数据。
   - **Step 2**: 在类元数据的方法表中，根据方法签名（名称+参数类型）查找对应方法的入口地址。

2. **静态绑定的优化**  
   - **非虚方法（如 `final`、`private`、静态方法）**：编译期间直接确定调用地址，无需运行时查找。
   - **虚方法（普通实例方法）**：运行时通过方法表动态分派。

---

### 三、内存交互图示
```java
// 示例代码
MyClass obj = new MyClass();
obj.method();
```

1. **堆中的 `obj` 对象**  
   - 包含实例变量和指向方法区 `MyClass` 元数据的类型指针。
2. **方法区中的 `MyClass` 元数据**  
   - 存储 `method()` 的字节码和方法表（含 `method()` 的入口地址）。
3. **调用过程**  
   - JVM 根据类型指针找到方法表，再通过方法表定位到 `method()` 的实际代码地址并执行。

---

### 四、关键区别与总结
| **存储位置** | **内容**                       | **访问方式**         |
| ------------ | ------------------------------ | -------------------- |
| 堆区         | 对象实例数据 + 类型指针        | 直接通过对象引用访问 |
| 方法区       | 类元数据（方法字节码、方法表） | 通过类型指针间接访问 |

- **堆区不保存方法地址**：仅通过类型指针与方法区建立联系。
- **方法区统一管理代码**：所有对象共享同一份类元数据，避免冗余存储。
- **高效动态绑定**：方法表机制兼顾性能与多态特性。

通过这种方式，Java 实现了对象与方法的解耦，既保证了内存效率，又支持了面向对象的核心特性（如继承和多态）。
