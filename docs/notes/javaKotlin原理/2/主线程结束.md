------

## ✅ 一、结论先说

> Java 主线程（`main()`）结束后，**JVM 是否退出取决于是否还有非守护线程（non-daemon thread）在运行。**
>
> - ✅ 如果只剩下 **守护线程（daemon thread）**，JVM 会立即退出；
> - 🚫 如果还有 **非守护线程（用户线程）** 存活，JVM 不会退出。

------

## 🧩 二、主线程与 JVM 生命周期的关系

启动 Java 程序时：

```bash
java MyApp
```

JVM 会：

1. 创建主线程（非守护线程）；
2. 调用 `MyApp.main(String[] args)`；
3. 执行完后，如果没有其他非守护线程存活，就退出。

------

## ⚙️ 三、举例说明

### ✅ 示例 1：主线程结束后 JVM 退出

```java
public class Test1 {
    public static void main(String[] args) {
        Thread t = new Thread(() -> {
            while (true) {
                System.out.println("Daemon running...");
                try { Thread.sleep(1000); } catch (InterruptedException e) {}
            }
        });
        t.setDaemon(true); // 守护线程
        t.start();

        System.out.println("Main end");
    }
}
```

输出：

```
Main end
```

然后程序立即退出（守护线程不会阻止 JVM 结束）。

------

### 🚫 示例 2：主线程结束后 JVM 不退出

```java
public class Test2 {
    public static void main(String[] args) {
        Thread t = new Thread(() -> {
            while (true) {
                System.out.println("Worker running...");
                try { Thread.sleep(1000); } catch (InterruptedException e) {}
            }
        });
        // t.setDaemon(false); 默认就是非守护线程
        t.start();

        System.out.println("Main end");
    }
}
```

输出：

```
Main end
Worker running...
Worker running...
...
```

此时主线程结束了，但 JVM **不会退出**，因为 `t` 是**非守护线程**。
 只有当 `t` 结束（正常返回或异常终止）后，JVM 才会退出。

------

## 🔍 四、守护线程（Daemon Thread）的特性

| 特性                 | 说明                             |
| -------------------- | -------------------------------- |
| 生命周期             | 随 JVM 生命周期结束而终止        |
| 关闭时机             | 当只剩守护线程时，JVM 自动退出   |
| 用途                 | GC、后台心跳、监控等后台任务     |
| 不保证执行完 finally | JVM 退出时守护线程可能被直接杀死 |

比如：

```java
Thread daemon = new Thread(() -> {
    try {
        Thread.sleep(3000);
        System.out.println("Daemon done");
    } finally {
        System.out.println("Cleanup code not guaranteed");
    }
});
daemon.setDaemon(true);
daemon.start();
```

如果主线程很快结束，JVM 会立即退出，
 即使守护线程还没到 finally 块，它也不会被执行完。

------

## ⚙️ 五、JVM 退出的三个条件

JVM 进程会在以下三种情况下退出：

| 触发方式                             | 描述                   |
| ------------------------------------ | ---------------------- |
| **主线程执行完，且无其他非守护线程** | 最常见                 |
| **显式调用 `System.exit(status)`**   | 立即退出，所有线程停止 |
| **JVM 被外部强制终止（如 kill -9）** | 直接终止，无清理       |

------

## 💡 六、一个经典误区：`Executors.newCachedThreadPool()` 默认是非守护线程！

```java
ExecutorService pool = Executors.newCachedThreadPool();
pool.submit(() -> { while(true) {} });
System.out.println("Main end");
```

⚠️ 程序永远不会退出，因为线程池里的线程是非守护线程。
 如果要让 JVM 能退出，应使用自定义线程工厂：

```java
ExecutorService pool = Executors.newCachedThreadPool(r -> {
    Thread t = new Thread(r);
    t.setDaemon(true);
    return t;
});
```

------

## 🧠 七、一句话总结

> Java 主线程结束后，
>
> - 若仍有 **非守护线程** → JVM 继续运行；
> - 若仅剩 **守护线程** → JVM 自动退出；
> - JVM 退出时不会等待守护线程执行完 finally 或 I/O 操作。

------

## 📘 八、面试高分版回答

> JVM 在启动时会创建主线程（非守护线程）执行 `main()`。
>  当主线程结束后，JVM 不会立即退出，它会检查所有线程状态：
>
> - 若仍存在非守护线程，JVM 继续运行；
> - 若仅剩守护线程，JVM 立即退出。
>    守护线程用于后台服务，如 GC、监控、缓存刷新等，
>    但 JVM 退出时不会保证它们执行完毕。
