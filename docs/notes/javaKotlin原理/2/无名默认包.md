我在当前项目下建了个ServiceLoader类，在当前项目下另外的包中无法import这个类，怎么解决？

IDE 报 “Cannot resolve symbol …” 99 % 不是 Java 语法问题，而是 **“编译器根本没找到你建的那个类”**。按下面顺序逐项排查，基本都能解决。

------------------------------------------------
1. 类必须 `public`  
   ```java
   package com.xxx.util;
   
   public class ServiceLoader {   // ← 少了 public 别的包就不可见
       ...
   }
   ```

2. 包名、目录、import 三对齐  
   - 文件物理路径：  
     `src/main/java/com/xxx/util/ServiceLoader.java`  
   - 源码里声明：  
     ```java
     package com.xxx.util;
     ```
   - 使用方 import：  
     ```java
     import com.xxx.util.ServiceLoader;
     ```

3. 别让 “自己写的” 把 “JDK 的” 盖住  
   JDK 自带 `java.util.ServiceLoader`。如果你也起同名类，又只写  
   ```java
   import java.util.ServiceLoader;   // 实际引的是 JDK 的
   ```
   就会看起来 “找不到自己那个”。  
   → 改个名字（`MyServiceLoader` / `ServiceLoaderImpl`）或者写全限定名：
   ```java
   com.xxx.util.ServiceLoader loader = new com.xxx.util.ServiceLoader();
   ```

4. 确认模块系统（JDK 9+）没有封包  
   如果用了 `module-info.java`：  
   ```java
   module myapp {
       exports com.xxx.util;   // 必须显式导出，否则别的包在模块外不可见
   }
   ```

5. IDE / 构建工具刷新  
   - IntelliJ：  
     `File → Invalidate Caches / Restart`  
     或 `Build → Rebuild Project`  
   - Maven / Gradle：  
     ```bash
     mvn clean compile
     # 或
     ./gradlew clean build
     ```

6. 多源码目录冲突  
   检查是否 **同时** 存在  
   `src/main/java/com/xxx/util/ServiceLoader.java`  
   `src/test/java/com/xxx/util/ServiceLoader.java`  
   且作用域交叉导致编译器放弃生成 class。

------------------------------------------------
一条命令快速验证  
```bash
# Linux / macOS
find . -name "ServiceLoader.java" -exec grep -H "public class" {} \;

# Windows
for /r %i in (ServiceLoader.java) do @findstr "public class" "%i"
```
看输出里有没有 `public class ServiceLoader` 以及路径是否和 import 语句一致。

------------------------------------------------
90 % 的场景都是 1.2.3 步就解决：  
**“public + 包名目录对齐 + 别和 JDK 同名”** 做完就能正常 `import`。

是Public的，而且我没起全类名，全类名直接是ServiceLoader

问题就出在 **“全类名只有 ServiceLoader”** —— 你没写 `package` 语句，于是 Java 把它放进了 **“无名默认包”**（default package）。  
**无名包里的类对任何有名包都不可 import**，这是 Java 的硬性语法限制，跟 public 不 public 无关。

------------------------------------------------
验证  
打开你的文件：

```java
// 最顶上没有 package 行
public class ServiceLoader { ... }
```

编译后目录：

```
classes/
└── ServiceLoader.class   // 类名就是 "ServiceLoader"，没有包前缀
```

在别的包（例如 `com.xxx.biz.XXX`）里写：

```java
import ServiceLoader;               // ❌ 编译错误
// 或
import com.xxx.ServiceLoader;       // ❌ 也不对，因为类根本不在任何包里
```

------------------------------------------------
解决办法（唯一出路）

1. 给类加一个有名字的包  
   把源码文件挪进对应目录，并在最顶部加 package 声明：

   ```java
   package com.xxx.util;
   
   public class ServiceLoader {
       ...
   }
   ```

   目录结构：

   ```
   src
   └── main
       └── java
           └── com
               └── xxx
                   └── util
                       └── ServiceLoader.java
   ```

2. 使用方正常 import

   ```java
   import com.xxx.util.ServiceLoader;
   ```

------------------------------------------------
小结  
- **默认包不能被别人 import**，这是语言规范，IDE 也救不了。  
- 只要把你的 `ServiceLoader` 放进 **有名包**，再按上面的目录/包名对齐，就能在任何其他包里 `import` 使用了。
