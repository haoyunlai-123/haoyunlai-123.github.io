```c++
struct oopDesc {
    markWord _mark;          // 锁状态、hash、GC年龄
    Klass* _metadata;        // 指向类模板的指针
    // 对齐
    // 实例字段（x, y）
    jint x;
    jint y;
};这个结构体中mark字段和metadata各占几字节？
```

非常好的问题 👍，这说明你已经把 Java 对象模型（oop layout）理解到 HotSpot C++ 源码层面了。
我们现在就以 HotSpot 的对象头实现（`oopDesc`）为基础，精确说明：
`markWord` 和 `Klass*` 各占多少字节、为什么占这么多、在不同位宽下有什么区别。

---

## 🧩 一、`oopDesc` 是 HotSpot 所有 Java 对象的基础结构

在 HotSpot 源码（`oop.hpp`）中：

```cpp
class oopDesc {
 private:
  volatile markWord _mark;
  union _metadata {
    Klass* _klass;
    narrowKlass _compressed_klass;
  } _metadata;
  ...
};
```

每个 Java 对象（包括数组）在堆中都以这个对象头开头。

---

## ⚙️ 二、对象头的两个主要字段

| 字段名                    | 作用                       | 位宽依赖                      |
| ---------------------- | ------------------------ | ------------------------- |
| `_mark` (`markWord`)   | 存储锁标志、哈希码、GC年龄、偏向锁状态等    | 固定 8 字节（64 位机器）           |
| `_metadata` (`Klass*`) | 指向类元信息（Klass）结构，用于知道对象类型 | 通常 4 字节（压缩类指针）或 8 字节（未压缩） |

---

## 🧠 三、`markWord` 大小：**8 字节**

`markWord` 是一个结构体封装，但底层本质是一个 64-bit union。
源码（`markWord.hpp`）：

```cpp
class markWord {
  uintptr_t _value;  // unsigned long long (64 bits)
  ...
};
```

在 64 位 HotSpot 中：

* `_value` 占 8 字节；
* 32 位 HotSpot 则占 4 字节。

📌 所以：

> * 在 64 位 JVM 中：`markWord` = **8 字节**
> * 在 32 位 JVM 中：`markWord` = **4 字节**

---

## 🧩 四、`Klass* _metadata` 大小：**4 或 8 字节**

### 两种情况：

| JVM 配置                            | `_metadata` 实际类型       | 大小       | 说明             |
| --------------------------------- | ---------------------- | -------- | -------------- |
| 默认开启 `UseCompressedClassPointers` | `narrowKlass` (32-bit) | **4 字节** | Klass 指针经过压缩存储 |
| 关闭压缩                              | `Klass*` (64-bit 指针)   | **8 字节** | 直接保存原生指针       |

HotSpot 默认在 64 位 JVM 中开启 “压缩类指针（Compressed Class Pointer）”，
因此大多数情况下 `_metadata` 只占 4 字节。

---

## 🧱 五、完整对象头布局总结（64-bit HotSpot）

| 字段                        | 默认压缩指针          | 未压缩指针     |
| ------------------------- | --------------- | --------- |
| markWord `_mark`          | 8 字节            | 8 字节      |
| Klass pointer `_metadata` | 4 字节            | 8 字节      |
| 对齐填充（padding）             | 4 字节（为了 8 字节对齐） | 无         |
| **小计**                    | **16 字节**       | **16 字节** |

✅ 也就是说：

> 绝大多数现代 JVM 上（64 位 + 压缩类指针启用），
> 对象头总大小是 **16 字节**，其中：
>
> * `_mark`：8 字节
> * `_metadata`：4 字节
> * 对齐填充：4 字节（保证对象起始地址是 8 字节对齐）

---

## 🧩 六、示例：以你的 `User` 对象为例

```cpp
struct oopDesc {
    markWord _mark;     // 8字节
    Klass* _metadata;   // 4字节 (压缩)
    int x;              // 4字节
    int y;              // 4字节
    // 对齐填充：4字节 (确保8字节对齐)
};
```

| 字段                    | 大小（字节）   |
| --------------------- | -------- |
| markWord              | 8        |
| metadata (compressed) | 4        |
| x                     | 4        |
| y                     | 4        |
| padding               | 4        |
| **合计**                | **24字节** |

所以你的示例对象在堆上占用 **24 字节**。

## ⚙️ 七、如何验证？

你可以运行以下命令：

```bash
java -XX:+PrintCompressedOopsMode -XX:+PrintClassLayout -XX:-UseTLAB -XX:-CompactFields
```

或者直接在代码中使用：

```java
System.out.println(VM.current().details());
System.out.println(ClassLayout.parseInstance(new User()).toPrintable());
```

输出中能看到：

```
mark: 8 bytes
class pointer: 4 bytes
instance fields: 8 bytes
alignment/padding: 4 bytes
Total: 24 bytes
```

## ✅ 八、总结

| 字段          | 类型                       | 默认大小（64位 + 压缩开启） | 未压缩时大小 |
| ----------- | ------------------------ | ---------------- | ------ |
| `_mark`     | `markWord`               | **8B**           | **8B** |
| `_metadata` | `narrowKlass` / `Klass*` | **4B**           | **8B** |
| 对齐填充        | —                        | 4B               | 0B     |

> 🧠 所以在 64 位 JVM 中，对象头固定 16 字节（默认压缩），
> `_mark` 8B + `_metadata` 4B + padding 4B。
