java中有个类继承了泛型类，并指定了具体类型，能在这个子类中通过反射拿到这个具体类型吗？

是的 ✅，**可以通过反射获取子类继承的泛型实参类型** —— 前提是子类在继承时明确写出了泛型的具体类型参数（没有被擦除为通配符或类型变量）。

------

### 📘 示例讲解

假设有如下父类与子类：

```java
class Base<T> { }

class User { }

class UserService extends Base<User> { }
```

我们希望在 `UserService` 运行时拿到它继承 `Base<User>` 时的具体类型参数 `User`。

------

### ✅ 获取泛型实参的正确方式

```java
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;

public class Main {
    public static void main(String[] args) {
        // 1️⃣ 获取父类的泛型信息
        Type type = UserService.class.getGenericSuperclass();

        // 2️⃣ 确认它是 ParameterizedType
        if (type instanceof ParameterizedType) {
            ParameterizedType pType = (ParameterizedType) type;

            // 3️⃣ 获取实际的类型参数
            Type actualType = pType.getActualTypeArguments()[0];
            System.out.println("泛型实参类型：" + actualType.getTypeName());
        }
    }
}
```

输出结果：

```
泛型实参类型：User
```

------

### ⚠️ 注意点

| 情况                                                     | 能否获取具体类型                                |
| -------------------------------------------------------- | ----------------------------------------------- |
| 子类明确写死泛型类型，如 `Base<User>`                    | ✅ 能拿到                                        |
| 子类仍为泛型，如 `class Service<T> extends Base<T>`      | ❌ 拿不到具体类型（只会拿到 `T`）                |
| 子类多层继承（如 `AdvancedService extends UserService`） | ⚠️ 需递归向上找第一个 `ParameterizedType` 的父类 |
| 匿名内部类，如 `new Base<User>(){}`                      | ✅ 一样可以拿到泛型实参（常用于 DAO 模板模式）   |

------

### 🌰 扩展示例：多层继承处理

```java
class Base<T> {}
class Mid<T> extends Base<T> {}
class Final extends Mid<User> {}

public class Demo {
    public static void main(String[] args) {
        Type type = Final.class.getGenericSuperclass(); // Mid<User>
        while (type != null) {
            if (type instanceof ParameterizedType) {
                Type actual = ((ParameterizedType) type).getActualTypeArguments()[0];
                System.out.println("最终类型: " + actual.getTypeName());
                break;
            }
            if (type instanceof Class) {
                type = ((Class<?>) type).getGenericSuperclass();
            }
        }
    }
}
```

输出：

```
最终类型: User
```

------

### 💡 总结

| 方法                                                         | 说明               |
| ------------------------------------------------------------ | ------------------ |
| `clazz.getGenericSuperclass()`                               | 获取父类的泛型签名 |
| `clazz.getGenericInterfaces()`                               | 获取接口的泛型签名 |
| `ParameterizedType.getActualTypeArguments()`                 | 获取泛型实参数组   |
| ⚠️ 泛型信息只在“**编译期已指定具体类型**”的情况下保留，否则会擦除 |                    |

是不是编译后子类泛型已经是具体的类型了，而不是Object?

✅ 非常好的问题！这个恰好触及 Java 泛型“**擦除 (Type Erasure)**”的核心机制。
 答案是：**取决于你看的层面**。编译后的字节码里，**类型参数确实被擦除了**（变成 `Object` 或上界类型），但**类的结构元信息（class metadata）中保留了泛型签名**，因此**可以通过反射读取出“泛型的声明信息”**，而不是运行时对象的真实类型。

下面我分两层来讲清楚这件事👇

------

## 🧩 一、字节码层面（运行时执行）

在 **JVM 执行时**，泛型信息是被擦除的。
 也就是说，以下两个类在字节码的“方法调用和字段访问层面”上是等价的：

```java
class Base<T> {
    T value;
    void set(T v) { value = v; }
    T get() { return value; }
}

class UserService extends Base<User> { }
```

编译后（用 `javap -c -v UserService.class` 看）：

```java
class Base {
    java.lang.Object value;
    void set(java.lang.Object);
    java.lang.Object get();
}
```

> 也就是说：**泛型参数 T 变成了 Object（或其上界）**，编译器在编译时会自动插入强制类型转换，保证类型安全。

这就是“泛型擦除”的含义。

------

## 🧠 二、类元信息层面（反射可见）

虽然 JVM 执行时看不到类型参数，但 **编译器会把泛型签名写进 class 文件的常量池中**（以 `Signature` 属性形式存储）。

比如 `UserService` 的 class 文件中会有这样一段签名：

```
Signature: #15                          // LBase<LUser;>;
```

也就是告诉反射系统：

> “这个类继承自 Base”。

因此当你用反射去看：

```java
Type t = UserService.class.getGenericSuperclass();
System.out.println(t);
```

输出的其实是：

```
Base<User>
```

这是因为反射读取的是 class 文件里的 **泛型签名信息**，而不是运行时对象里的字段类型。

------

## ⚙️ 三、总结对比表

| 层面                                | 泛型信息                  | 是否能获取具体类型 | 示例         |
| ----------------------------------- | ------------------------- | ------------------ | ------------ |
| **字节码执行层面 (JVM runtime)**    | 被擦除成 `Object` 或上界  | ❌ 拿不到           | `T → Object` |
| **反射元信息层面 (Class metadata)** | 保留泛型签名（Signature） | ✅ 可以读取         | `Base<User>` |
| **运行时对象实例 (实际对象)**       | 没有泛型类型参数信息      | ❌ 拿不到           | `Object`     |

------

## 🔍 直观验证

```java

System.out.println(base.getClass().getGenericSuperclass());
```

输出：

```
null
```

因为 `Base` 自己是泛型类，没指定父类泛型类型。

而：

```java

System.out.println(service.getClass().getGenericSuperclass());
```

输出：

```
Base<User>
```

因为编译器在 class 文件元信息里写明了 `UserService extends Base<User>`。

------

## ✅ 结论

> **是的，编译后的子类在反射元信息中记录了具体类型（User），但在字节码执行时泛型类型已经被擦除为 Object（或其上界）**。
>
> 因此：
>
> - **通过反射能拿到类型信息** ✅（读取 class 元信息）。
> - **通过运行时实例拿不到** ❌（执行层面类型擦除）。

## 🌱 一、前置知识

假设我们有这样三层继承：

```java
class Base<T> {}

class Mid<T> extends Base<T> {}

class User {}

class Final extends Mid<User> {}
```

目标：
 👉 在 `Demo.main()` 中，通过反射从 `Final.class` 找到它最终的泛型实参 —— `User`。

因为 `Final` 继承了 `Mid<User>`，`Mid` 又继承了 `Base<T>`，
 **泛型实参信息可能在多层父类中才被明确指定。**

------

## 🧩 二、逐行解析这段代码

```java
Type type = Final.class.getGenericSuperclass(); // Mid<User>
```

### ✅ 第 1 步：拿到直接父类的类型信息

- `Final.class.getGenericSuperclass()` 会返回 `Final` 的直接父类的“类型”信息。

- 对 `Final` 来说，它的父类是 `Mid<User>`。

- 所以此时 `type` 是一个 **ParameterizedType**（即“带泛型参数的类型”），其字符串表现为：

  ```
  Mid<User>
  ```

------

```java
while (type != null) {
```

### ✅ 第 2 步：循环向上遍历父类链

- 这是为了**防止泛型实参不在当前层，而在更上层**。
- 例如：`Final → Mid<T> → Base<User>` 时，需要往上走才能找到真正的泛型参数。

------

```java
if (type instanceof ParameterizedType) {
    ParameterizedType pType = (ParameterizedType) type;
    Type actual = pType.getActualTypeArguments()[0];
    System.out.println("最终类型: " + actual.getTypeName());
    break;
}
```

### ✅ 第 3 步：如果当前父类是“带泛型参数的类型”，就解析它

- `ParameterizedType` 表示像 `Mid<User>`、`Base<String>` 这种“明确指定了泛型实参”的类型。
- `getActualTypeArguments()` 会返回一个数组，比如：
  - 对 `Mid<User>` 来说，就是 `[User]`；
  - 对 `Map<String, Integer>` 来说，就是 `[String, Integer]`。
- 取 `[0]` 就是第一个泛型实参。
- `getTypeName()` 返回的是它的名字（比如 `"User"`）。

> ⚙️ 举例：
>
> ```java
> ParameterizedType p = (ParameterizedType) Final.class.getGenericSuperclass();
> System.out.println(p.getActualTypeArguments()[0]); // class User
> ```

------

```java
if (type instanceof Class) {
    type = ((Class<?>) type).getGenericSuperclass();
}
```

### ✅ 第 4 步：如果当前层不是 ParameterizedType（即没带泛型），就继续往上查找父类

- 某些中间类定义为普通类（比如 `class Mid extends Base<User>`），
   那 `Mid.class.getGenericSuperclass()` 可能直接是 `Base<User>`。
- 所以要不断 `getGenericSuperclass()`，直到遇到一个 `ParameterizedType`。

> 举个更复杂的例子：
>
> ```java
> class Base<T> {}
> class Mid extends Base<String> {}
> class Final extends Mid {}
> ```
>
> `Final.class.getGenericSuperclass()` 得到的只是 `class Mid`（不是 ParameterizedType），
>  所以需要再往上取一次：`Mid.class.getGenericSuperclass()` 才能拿到 `Base<String>`。
