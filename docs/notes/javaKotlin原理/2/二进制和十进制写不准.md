写不准（循环二进制）： 任何十进制小数只要分母含非 2 的质因子就会无限循环。这个怎么理解

一句话：  
“小数在二进制里能不能写准，只看它的**最简分数**的分母；分母里一旦冒出 **2 以外的质因子**，二进制表示就必成无限循环，再也写不完。”

------------------------------------------------
把道理拆成 4 步看：

1. 二进制小数的位置值  
   小数点后第 k 位权值是 2⁻ᵏ，因此任何有限位二进制小数展开后，总可以写成  
   n / 2ᵐ   （n 整数，m ≥ 0）

2. 十进制小数 → 最简分数  
   拿 0.6 举例：  
   0.6 = 6/10 = 3/5  
   分母 5 含有质因子 **5** ≠ 2，于是它**不可能**等于任何 n / 2ᵐ。

3. 长除法看循环  
   把 0.6 不停地“×2 取整”：

   ```
   0.6 × 2 = 1.2   → 1
   0.2 × 2 = 0.4   → 0
   0.4 × 2 = 0.8   → 0
   0.8 × 2 = 1.6   → 1
   0.6 × 2 = 1.2   → 1   （回到开头，循环开始）
   ```

   得到 0.1001100110011…₂  → **无限循环 0011**。

4. 反过来验证“纯 2 的幂”就能写准  
   0.375 = 3/8 = 3/2³  
   分母只有 2，于是有限位：  
   0.375 ×2=0.75 0  
   0.75 ×2=1.5  1  
   0.5 ×2=1.0   1        → 0.011₂，**终止**。

------------------------------------------------
通用判据  
给定十进制小数，化到最简分数 **p/q**：

- q 的质因子 **只有 2** → 二进制**有限**  
- q 出现 **任何非 2 质因子（3,5,7,11…）** → 二进制**无限循环**

因此 0.1(1/10)、0.2(1/5)、0.3(3/10)、0.7(7/10)… 统统写不准；0.5、0.25、0.125、0.6875(11/16) 都能精确存成 float/double。

写不准： 分母含 非 2 或 5 的质因子 的分数，如 1/3、1/7、1/12 … 0.1 可以精确保存，但 1/3 依旧不行。这个怎么理解

把“写不准”这句话拆成两步看，就明白了：

1. 先分清“**哪种进制的小数**”  
   - 你举的例子是 **十进制小数** 在 **十进制浮点**（decimal32/64/128）里能不能写准。  
   - 前面讲的“分母含非 2 因子就循环”是 **十进制小数** 放到 **二进制浮点** (float/double) 里的规律。  
   两种场景判据不同，别混到一起。

2. 十进制浮点的判据——“分母只能含 2 和 5”  
   十进制浮点的小数位权是 10⁻ᵏ = (2·5)⁻ᵏ，因此**任何有限位十进制小数**最终都能写成  
   n / (2ᵃ·5ᵇ)  
   只要最简分数的分母**跑出 2 或 5 之外的质因子**，就永远凑不成这种形式，于是十进制表示也无限循环。

举例验证：

| 分数  | 最简分母   | 十进制小数      | 能否在 decimal128 写准 | 原因         |
| ----- | ---------- | --------------- | ---------------------- | ------------ |
| 1/3   | 3          | 0.3333…         | ❌                      | 出现质因子 3 |
| 1/7   | 7          | 0.142857142857… | ❌                      | 出现质因子 7 |
| 1/12  | 12 = 2²·3  | 0.08333…        | ❌                      | 出现质因子 3 |
| 0.1   | 10 = 2·5   | 0.1             | ✅                      | 只有 2 和 5  |
| 0.025 | 40 = 2³·5  | 0.025           | ✅                      | 只有 2 和 5  |
| 1/30  | 30 = 2·3·5 | 0.0333…         | ❌                      | 出现质因子 3 |

结论（针对 **十进制浮点**）：

- 把十进制小数化成最简分数 **p/q**  
- q 的质因子**只能有 2 或 5** → 可以**精确**保存  
- q 一旦含 **3, 7, 11, …** → 十进制下**无限循环**，decimal32/64/128 也只能舍入存储，和 1/3 永远写不准是一个道理。

1/12在二进制中也不能写准吧，因为有3质因子
完全正确。

1/12 化成最简分数是 1/(2²·3)。  
分母里出现了 **3** —— 2 以外的质因子 —— 因此它在**二进制**下必然无限循环，写不准。

用“×2 取整”长除法跑一遍：

```
0.083333... (十进制)
×2  0.1666...  0
×2  0.3333...  0
×2  0.6666...  0
×2  1.3333...  1
×2  0.6666...  0   ← 回到第 4 行，循环开始
```

得到二进制小数：

```
0.0001010101010101...₂   （循环节 "01"）
```

所以无论在 **float** 还是 **double** 里，1/12 都只能保存成舍入后的近似值，永远不可能像 0.125 那样精确终止。

RoundingMode.HALF_EVEN这是什么保留规则

`RoundingMode.HALF_EVEN` 是 Java（BigDecimal / MathContext）以及 IEEE-754 标准里定义的 **“银行家舍入”**（Banker’s rounding），也叫 **“向最接近偶数舍入”** 或 **“四舍六入五成双”**。

规则一句话  
**“四舍六入，五看奇偶”**——  
当舍弃部分 **恰好等于 0.5** 时，**让保留位变成偶数**；其余情况照常四舍六入。

举例（保留 1 位小数）：

| 原始值 | 舍弃部分                          | 结果 | 理由                       |
| ------ | --------------------------------- | ---- | -------------------------- |
| 1.24   | 0.04 < 0.5                        | 1.2  | 四舍                       |
| 1.26   | 0.06 > 0.5                        | 1.3  | 六入                       |
| 1.25   | 0.5，且保留位 2 是偶数 → **不动** | 1.2  | 五成双                     |
| 1.35   | 0.5，且保留位 3 是奇数 → **进位** | 1.4  | 五成双                     |
| 1.450  | 0.50，保留位 4 偶数               | 1.4  | 同上                       |
| −1.25  | 0.5，保留位 2 偶数                | −1.2 | 符号位无关，只看绝对值奇偶 |

优点  
- 大量累加时 **偏差趋近于 0**（普通“四舍五入”会系统性地向上偏）。  
- 因此财务、统计、科学计算中被默认采用。

记忆口诀  
**四舍六入五成双，五后非零则进位，五后皆零看奇偶。**
