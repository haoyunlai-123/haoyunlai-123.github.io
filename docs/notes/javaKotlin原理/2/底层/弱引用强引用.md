java中弱引用和强引用是什么意思？

在 Java 中，**强引用（Strong Reference）**和**弱引用（Weak Reference）**是两种不同强度的对象引用方式，它们直接影响对象何时可以被垃圾回收（GC）。

---

### ✅ 强引用（Strong Reference）
这是最常见的引用类型。

```java
Object obj = new Object();
```

- **只要强引用存在，垃圾回收器永远不会回收这个对象。**
- 即使内存紧张，JVM 也宁愿抛出 `OutOfMemoryError`，也不会回收被强引用指向的对象。

---

### ✅ 弱引用（Weak Reference）
通过 `java.lang.ref.WeakReference` 类实现。

```java
WeakReference<Object> weakRef = new WeakReference<>(new Object());
```

- **弱引用不会阻止垃圾回收。**
- 只要垃圾回收器运行，**无论内存是否充足**，**只被弱引用指向的对象都会被回收**。
- 常用于实现**缓存**或**元数据映射**，比如 `WeakHashMap`。

---

### 🔍 举个例子对比：

```java
Object strong = new Object(); // 强引用
WeakReference<Object> weak = new WeakReference<>(new Object()); // 弱引用

System.gc(); // 建议 JVM 执行垃圾回收

System.out.println(strong); // 一定不为 null
System.out.println(weak.get()); // 可能为 null（被回收了）
```

---

### ✅ 总结一句话：
- **强引用：只要引用在，对象永不被回收。**
- **弱引用：GC 一来，对象就可能被回收。**

什么原理？Object对象指针不是还在strong中保存吗？怎么回收掉weak中的Object引用的？

你问得非常关键，这里确实容易混淆。我们来**从 JVM 的视角**把这个问题彻底讲清楚。

---

### ✅ 你提到的核心疑问是：

> `strong` 和 `weak` 不是都指向一个 `Object` 吗？`strong` 还在，怎么 `weak.get()` 就变成 `null` 了？

---

### ✅ 关键前提：**你这两个引用指向的不是同一个对象！**

```java
Object strong = new Object(); // 对象 A
WeakReference<Object> weak = new WeakReference<>(new Object()); // 对象 B
```

- `strong` 指向的是**对象 A**
- `weak` 内部包装的是**对象 B**

这两个对象**没有任何关系**，只是你写代码的时候看起来“都像 Object”。

---

### ✅ 正确的对比方式（指向同一个对象）：

```java
Object obj = new Object(); // 只有一个对象
Object strong = obj; // 强引用
WeakReference<Object> weak = new WeakReference<>(obj); // 弱引用

System.gc();

System.out.println(strong); // 不会 null
System.out.println(weak.get()); // 也不会 null，因为还有强引用 strong 指向它
```

---

### ✅ 只有弱引用时才会被回收：

```java
WeakReference<Object> weak = new WeakReference<>(new Object());
// 此时 new Object() 只有 weak 这一个弱引用指向它

System.gc();

System.out.println(weak.get()); // 大概率是 null
```

---

### ✅ 总结一句话：

> **JVM 判断一个对象是否可回收，是看它是否还**强可达（strongly reachable）**。  
> 只要有一个强引用链还连着它，就不会回收。  
> 只有弱引用、软引用或没有引用时，才可能被回收。**

---

### ✅ 图示理解（简化）：

```
[栈变量 strong] → [对象 A] ← [WeakReference 内部指针]
```

- 只要 `strong` 还在，**对象 A 就是强可达**，**不会被回收**。
- 当 `strong = null;` 后，**对象 A 只剩下弱引用**，**下次 GC 就可能回收**。
