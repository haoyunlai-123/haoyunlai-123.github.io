非常好 👍 这段代码的报错其实非常典型：

> ❌ 报错内容类似于：
> `Cannot reference a non-static field from a static context`
> 或
> `Cannot find symbol: variable registry`

---

## 🔍 报错原因解析

在这一行：

```java
PrototypeRegistry.registry
```

你写在了一个 **静态内部类（`Inner`）** 的静态成员定义部分。
而此时：

```java
public static final Map<String, Document> registry;
```

虽然 `registry` 被声明为 `static`，
但 **你还没有给它赋值（初始化）**，也**没有在类加载时完成定义**，
所以在 `Inner` 的静态块中直接访问会报「找不到符号」或「非法前向引用」。

---

## 🧠 关键点：静态成员的“前向引用限制”

在 Java 中，类的静态成员（包括字段和静态内部类）是**按出现顺序**加载和初始化的。

即：

* 当 JVM 解析 `PrototypeRegistry` 的字节码时，
  会先“看到”字段 `registry` 的声明，但**此时还没有完成初始化**；
* 然后它看到了 `Inner` 静态内部类；
* 而你在 `Inner` 内部立即引用了 `PrototypeRegistry.registry`，
  此时 **`registry` 尚未初始化**（甚至没有定义完成），
  属于“非法前向引用”。

---

## ✅ 正确写法 1：在定义时初始化 `registry`

直接给 `registry` 一个初始值（通常是空 `HashMap`）：

```java
public class PrototypeRegistry {

    public static final Map<String, Document> registry = new HashMap<>();

    private static class Inner {
        static final PrototypeRegistry prototypeRegistry = new PrototypeRegistry();
        // ✅ 现在可以正常引用 registry 了
    }

    public static PrototypeRegistry getPrototypeRegistry() {
        return Inner.prototypeRegistry;
    }

    public Document getDocument(String id) {
        return registry.get(id);
    }
}
```

✅ 这样写不会报错，因为：

* `registry` 在类加载时就完成初始化；
* 静态内部类 `Inner` 在首次使用时才被加载（懒加载特性）；
* 引用是合法的。

---

## ✅ 正确写法 2：如果希望延迟初始化

如果你想让 `registry` 在类加载时保持空引用，之后再由构造器或静态块赋值，可以：

```java
public class PrototypeRegistry {

    public static final Map<String, Document> registry;

    static {
        registry = new HashMap<>();
    }

    private static class Inner {
        static final PrototypeRegistry prototypeRegistry = new PrototypeRegistry();
        // ✅ 此时 static 块已经执行完，可以安全访问 registry
    }

    public static PrototypeRegistry getPrototypeRegistry() {
        return Inner.prototypeRegistry;
    }

    public Document getDocument(String id) {
        return registry.get(id);
    }
}
```

## 🧠 额外说明：你的类是想实现“原型注册表（Prototype Registry）”模式

在这个设计模式中：

* `PrototypeRegistry` 是一个**单例类**；
* 内部维护一个 `Map<String, Document>`；
* 外部可以通过 id 获取对应的 Document 原型，然后 `.clone()` 一个副本。

示例实现（完整可运行）：

```java
public class PrototypeRegistry {

    private static final Map<String, Document> registry = new HashMap<>();

    private static class Inner {
        static final PrototypeRegistry INSTANCE = new PrototypeRegistry();
    }

    public static PrototypeRegistry getInstance() {
        return Inner.INSTANCE;
    }

    public void register(String id, Document doc) {
        registry.put(id, doc);
    }

    public Document getDocument(String id) {
        Document doc = registry.get(id);
        return doc == null ? null : doc.clone();
    }
}
```

---

✅ **总结一句话：**

> 你那一行报错是因为在静态内部类定义阶段访问了还未初始化的静态成员（前向引用）。
> 解决方法：
>
> * 给 `registry` 立即初始化；或
> * 用静态初始化块赋值。
