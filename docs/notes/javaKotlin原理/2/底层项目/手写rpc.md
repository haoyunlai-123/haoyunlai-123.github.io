给你挑一个**既底层又硬核、还能写在简历上“眼前一亮”**的练手项目：**「手写迷你分布式 RPC 框架」**  
（对标 Dubbo 核心，但代码量可控，面试可讲清每一行）

------------------------------------------------
项目名：「MiniRPC」  
一句话定位：基于 Netty + Kryo + ZooKeeper 实现的 **分布式 RPC 框架**，涵盖 **服务注册/发现、负载均衡、异步调用、字节码生成、心跳保活、优雅关机** 等核心机制，**代码量 3k 行以内**，**单模块即可运行**。

------------------------------------------------
为什么选它
------------------------------------------------
1. **纯底层技术栈**  
   - 传输：Netty（零拷贝、Reactor）  
   - 序列化：Kryo（字节级压缩）  
   - 注册中心：ZooKeeper / etcd（看兴趣）  
   - 代理：Javassist 字节码生成（对标 Dubbo Proxy）  
   - 负载均衡：Random、WeightRoundRobin、一致性 Hash  
   - 容错：FailFast、FailOver、熔断计数器  
   **全程无 Spring**，**只依赖 Netty + Kryo + ZK 客户端**，**真正自己掌控生命周期**。

2. **面试爆点**  
   - 能讲 **“如何自己拼字节码代替 JDK Proxy”**  
   - 能讲 **“Netty  pipelines 怎么编排心跳、编解码、限流”**  
   - 能讲 **“ZK 临时节点怎么做服务上下线感知”**  
   - 能讲 **“异步调用 Future → CompletableFuture 链路”**  
   - 能讲 **“优雅关机：EventLoop 优雅关闭 + 正在处理请求计数器”**

3. **文档 & 教程现成**  
   开源社区有**完整实现 + 图文教程**可参考，但**代码量极简**，**可自己重写/二次改造**：
   - [guide-rpc-framework（5.8k star）]  
     文章手把手带你一行行写完，**总目录 40+ 篇**，**从 0 到压测**全流程。

4. **可扩展成“分布式”亮点**  
   - 加 **SPI 机制**（对标 Dubbo 扩展点）  
   - 加 **自定义协议**（魔数、版本、序列化方式、压缩位）  
   - 加 **Metrics**（QPS、RT、失败率，对接 Prometheus）  
   - 加 **链路追踪**（TraceId 透传，对标 Sleuth）  
   - 加 **熔断降级**（滑动窗口、错误率阈值）  
   **每多一个扩展，简历就多一个技术关键词**。

------------------------------------------------
技术路线（6 周节奏示例）
------------------------------------------------
| 周   | 里程碑                  | 输出                         |
| ---- | ----------------------- | ---------------------------- |
| 1    | **单体 Netty 通信**     | EchoServer + 长度帧解码      |
| 2    | **自定义协议 + 编解码** | 魔数、版本、序列化算法位     |
| 3    | **服务注册/发现**       | ZK 临时节点、监听上下线      |
| 4    | **代理层 + 负载均衡**   | Javassist 生成接口实现类     |
| 5    | **异步调用 + 容错**     | CompletableFuture + FailOver |
| 6    | **优雅关机 + 压测**     | JMH 压测、Grafana 看板       |
