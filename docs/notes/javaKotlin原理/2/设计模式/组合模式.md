介绍下组合模式并举几个例子

非常好 👍！你现在学到的是结构型设计模式中的一个非常重要的模式 ——
👉 **组合模式（Composite Pattern）**。

这个模式的核心思想极其优雅：

> 让用户**以统一的方式对待单个对象和组合对象**。
> （也就是“树形结构的统一表示与操作”）

---

## 🧩 一、定义与核心思想

**组合模式（Composite Pattern）**
是一种用于处理**树形层次结构**的结构型设计模式。

> 它让你把**对象组合成树形结构**来表示“整体/部分”的层次关系，
> 并使得客户端可以**用一致的方式对待单个对象和组合对象**。

---

### 🧠 一句话理解

> “树上每个节点，无论是叶子还是枝干，外部都用同一个接口操作。”

比如文件系统：

* 文件夹里可以包含文件，也可以包含其他文件夹；
* 无论是“文件夹”还是“文件”，对外都可以 `show()`。

这就体现了组合模式。

## ⚙️ 三、示例一：文件系统（最经典例子）

我们来模拟一个简单的文件系统，文件夹中可以包含文件或其他文件夹。

```java
import java.util.*;

// 抽象组件
interface FileSystemNode {
    void show(String prefix);
}

// 叶子节点：文件
class FileNode implements FileSystemNode {
    private final String name;
    public FileNode(String name) { this.name = name; }

    @Override
    public void show(String prefix) {
        System.out.println(prefix + "📄 " + name);
    }
}

// 容器节点：文件夹
class FolderNode implements FileSystemNode {
    private final String name;
    private final List<FileSystemNode> children = new ArrayList<>();

    public FolderNode(String name) { this.name = name; }

    public void add(FileSystemNode node) {
        children.add(node);
    }

    @Override
    public void show(String prefix) {
        System.out.println(prefix + "📁 " + name);
        for (FileSystemNode child : children) {
            child.show(prefix + "    ");
        }
    }
}

// 客户端
public class CompositeDemo {
    public static void main(String[] args) {
        FolderNode root = new FolderNode("root");
        FolderNode src = new FolderNode("src");
        FolderNode assets = new FolderNode("assets");

        src.add(new FileNode("Main.java"));
        src.add(new FileNode("Utils.java"));
        assets.add(new FileNode("logo.png"));
        assets.add(new FileNode("config.json"));

        root.add(src);
        root.add(assets);
        root.add(new FileNode("README.md"));

        root.show("");
    }
}
```

✅ 输出：

```
📁 root
    📁 src
        📄 Main.java
        📄 Utils.java
    📁 assets
        📄 logo.png
        📄 config.json
    📄 README.md
```

💡 客户端不区分 `FileNode` 和 `FolderNode`，统一调用 `show()` 即可。

## 🧩 四、示例二：公司组织架构

公司组织层次：

* 总公司

  * 分公司

    * 部门

      * 员工

无论是“员工”还是“部门”，都实现统一的接口。

```java
import java.util.*;

interface OrganizationUnit {
    void show(int indent);
}

class Employee implements OrganizationUnit {
    private final String name;
    public Employee(String name) { this.name = name; }

    public void show(int indent) {
        System.out.println(" ".repeat(indent) + "👤 员工: " + name);
    }
}

class Department implements OrganizationUnit {
    private final String name;
    private final List<OrganizationUnit> members = new ArrayList<>();

    public Department(String name) { this.name = name; }

    public void add(OrganizationUnit unit) { members.add(unit); }

    public void show(int indent) {
        System.out.println(" ".repeat(indent) + "🏢 部门: " + name);
        for (OrganizationUnit u : members)
            u.show(indent + 4);
    }
}

// 客户端
public class CompanyDemo {
    public static void main(String[] args) {
        Department head = new Department("总部");
        Department branch = new Department("南京分部");
        Department devDept = new Department("研发部");

        devDept.add(new Employee("Alice"));
        devDept.add(new Employee("Bob"));

        branch.add(devDept);
        branch.add(new Employee("BranchManager"));
        head.add(branch);

        head.show(0);
    }
}
```

✅ 输出：

```
🏢 部门: 总部
    🏢 部门: 南京分部
        🏢 部门: 研发部
            👤 员工: Alice
            👤 员工: Bob
        👤 员工: BranchManager
```

💡 客户端不用区分部门还是员工，统一用 `OrganizationUnit.show()`。

## 💡 五、示例三：UI 组件树（真实 GUI 场景）

图形界面中，界面元素（Button、Panel、Label）
就是典型的“组合模式”结构：

```java
abstract class UIComponent {
    abstract void render();
}

class Button extends UIComponent {
    void render() { System.out.println("绘制按钮"); }
}

class Label extends UIComponent {
    void render() { System.out.println("绘制标签"); }
}

class Panel extends UIComponent {
    List<UIComponent> children = new ArrayList<>();
    void add(UIComponent c) { children.add(c); }

    void render() {
        System.out.println("绘制面板");
        for (UIComponent c : children) {
            c.render();
        }
    }
}

// 客户端
public class UIDemo {
    public static void main(String[] args) {
        Panel panel = new Panel();
        panel.add(new Button());
        panel.add(new Label());
        Panel inner = new Panel();
        inner.add(new Button());
        panel.add(inner);
        panel.render();
    }
}
```

✅ 输出：

```
绘制面板
绘制按钮
绘制标签
绘制面板
绘制按钮
```

📘 这就是 Swing / JavaFX / Android UI 树的基本结构思想。

## 🧠 六、模式要点总结

| 角色          | 说明                                               |
| ------------- | -------------------------------------------------- |
| **Component** | 抽象组件，定义公共操作（如 `show()` / `render()`） |
| **Leaf**      | 叶子节点，无子对象（文件、员工）                   |
| **Composite** | 容器节点，包含多个 `Component` 子对象              |
| **Client**    | 使用统一接口操作整棵树，不关心节点类型             |

---

## ⚙️ 七、优缺点分析

| 优点       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| ✅ 统一接口 | 客户端可用一致方式操作单个或组合对象                         |
| ✅ 结构清晰 | 模拟树状结构（文件系统、组织结构等）                         |
| ✅ 易扩展   | 新增节点类型无需修改现有代码                                 |
| ⚠️ 缺点     | 不同节点逻辑差异大时可能增加复杂度（比如叶子不该支持 `add()` 但仍要继承） |

---

## 🧾 八、在框架中的应用

| 场景                       | 框架 / 实现                   |
| -------------------------- | ----------------------------- |
| GUI 组件树                 | Swing、JavaFX、Android View   |
| HTML DOM                   | 节点树结构统一接口            |
| 文件系统 API               | `File` 类即可能表示文件或目录 |
| 游戏引擎场景树             | Unity、Godot 的节点系统       |
| Spring BeanDefinition 结构 | 也是一种组合模型的配置树      |

---

## ✅ 九、一句话总结

> 🧩 **组合模式（Composite Pattern）**
> 让客户端以一致的方式对待“单个对象”和“对象组合”，
> 它是处理树形结构、层级关系最优雅的模式之一。
>
> 🌳 “让部分和整体拥有一致的接口。”
