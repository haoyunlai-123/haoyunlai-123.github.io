非常好 👍！
 你现在来到行为型模式中非常基础、但**几乎所有语言标准库都用到**的一个模式 ——
 👉 **迭代器模式（Iterator Pattern）**。

它是集合框架、流式 API、甚至数据库游标（cursor）的基础。
 理解这个模式，你就能彻底搞懂 `for-each`、`Iterator`、`Iterable`、`Stream` 的底层逻辑。

------

## 🧩 一、定义与核心思想

**迭代器模式（Iterator Pattern）**
 提供一种统一的方式来**顺序访问聚合对象中的元素**，
 而**不暴露该对象的内部结构**。

> 换句话说：
>  “让你能遍历一个集合，却不用关心它的内部是数组、链表还是树。”

------

### 🧠 一句话理解

> “把遍历逻辑封装成对象（迭代器），
>  使用者只管取下一个，不关心底层实现。”

------

## ⚙️ 三、示例一：手动实现一个简单的迭代器

假设我们有一个自定义的集合 `BookCollection`，
 我们希望能像标准集合一样遍历它。

```java
// 迭代器接口
interface Iterator<T> {
    boolean hasNext();
    T next();
}

// 聚合接口
interface Aggregate<T> {
    Iterator<T> createIterator();
}

// 具体聚合类
class BookCollection implements Aggregate<String> {
    private List<String> books = new ArrayList<>();

    public void addBook(String book) {
        books.add(book);
    }

    public Iterator<String> createIterator() {
        return new BookIterator();
    }

    // 内部类：具体迭代器
    private class BookIterator implements Iterator<String> {
        private int index = 0;

        public boolean hasNext() {
            return index < books.size();
        }

        public String next() {
            return books.get(index++);
        }
    }
}

// 客户端
public class IteratorDemo {
    public static void main(String[] args) {
        BookCollection collection = new BookCollection();
        collection.addBook("Design Patterns");
        collection.addBook("Effective Java");
        collection.addBook("Clean Code");

        Iterator<String> it = collection.createIterator();
        while (it.hasNext()) {
            System.out.println("📘 " + it.next());
        }
    }
}
```

✅ 输出：

```
📘 Design Patterns
📘 Effective Java
📘 Clean Code
```

💡 优点：

- 客户端完全不知道底层是 `ArrayList`；
- 可替换为数组、链表等实现；
- 符合单一职责原则（遍历逻辑独立于集合逻辑）。

## 💡 四、示例二：反向迭代器（扩展）

迭代器可以扩展为**不同的遍历方式**（正序、倒序、步进等）。

```java
class ReverseIterator<T> implements Iterator<T> {
    private List<T> list;
    private int index;

    public ReverseIterator(List<T> list) {
        this.list = list;
        this.index = list.size() - 1;
    }

    public boolean hasNext() {
        return index >= 0;
    }

    public T next() {
        return list.get(index--);
    }
}

public class ReverseIteratorDemo {
    public static void main(String[] args) {
        List<String> list = List.of("A", "B", "C");
        Iterator<String> it = new ReverseIterator<>(list);
        while (it.hasNext()) {
            System.out.println(it.next());
        }
    }
}
```

✅ 输出：

```
C
B
A
```

💡 这说明：
 **迭代器模式的强大之处在于“可以定义多种遍历策略”而不影响集合结构。**

## ⚙️ 五、示例三：JDK 内置迭代器（实际应用）

在 Java 中，几乎所有集合都实现了：

```java
public interface Iterable<T> {
    Iterator<T> iterator();
}
```

所以可以写：

```java
List<String> list = List.of("A", "B", "C");
for (String s : list) {
    System.out.println(s);
}
```

等价于：

```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}
```

💡 `for-each` 语法糖实际上就是迭代器模式的体现。

## ⚙️ 六、示例四：数据库游标（Cursor）也是迭代器！

比如 JDBC 的 `ResultSet`：

```java
ResultSet rs = stmt.executeQuery("SELECT name FROM users");
while (rs.next()) {
    System.out.println(rs.getString("name"));
}
```

✅ `ResultSet` 内部就实现了迭代器逻辑：

- `next()` → 移动到下一个记录；
- `getXXX()` → 读取当前行内容。

💡 它隐藏了底层网络、分页、缓存细节，让你像遍历集合一样遍历数据库结果。

## 🧠 七、迭代器模式的本质

> **迭代器模式的本质是“封装遍历逻辑”。**

它把遍历操作从聚合对象中抽离出来，让两者职责清晰：

- **聚合类（Aggregate）** 负责存储元素；
- **迭代器类（Iterator）** 负责遍历逻辑；
- **客户端** 通过迭代器访问，而不依赖具体集合结构。

------

## ⚙️ 八、优缺点总结

| 优点           | 说明                         |
| -------------- | ---------------------------- |
| ✅ 统一访问接口 | 不同集合都可统一遍历         |
| ✅ 封装内部结构 | 遍历时无需关心底层实现       |
| ✅ 易于扩展     | 可添加不同遍历方式           |
| ⚠️ 缺点         | 对象数量多；简单场景显得啰嗦 |

>
> 🧠 “让遍历与存储分离，访问方式更优雅。”
