介绍下备忘录模式并举几个例子

非常棒 👍！
你现在学到的是行为型模式中的一个非常“现实”、
也非常常见的设计模式 ——
👉 **备忘录模式（Memento Pattern）**。

它是实现 **“撤销 / 回滚 / 历史记录 / 快照”** 功能的关键机制。
无论是 IDE、游戏存档、事务回滚、浏览器后退、文本编辑器 undo，
背后都是备忘录模式在工作。

---

## 🧩 一、定义与核心思想

**备忘录模式（Memento Pattern）**
在不破坏封装的前提下，捕获一个对象的内部状态，
并在该对象之外保存这个状态，
这样以后就可以将对象恢复到原先的状态。

> 换句话说：
> “快照 + 回滚机制” 的通用实现。

---

### 🧠 一句话理解

> “在不暴露内部实现的前提下，记录对象的历史状态。”

---

## 🧱 二、结构图

```
┌──────────────────────┐
│     Originator        │ ← 发起者，负责创建/恢复备忘录
│──────────────────────│
│ + createMemento()     │
│ + restoreMemento(m)   │
└────────┬─────────────┘
         │
┌────────▼──────────────┐
│      Memento           │ ← 备忘录对象（封装状态）
│───────────────────────│
│ + getState()           │
└────────┬──────────────┘
         │
┌────────▼──────────────┐
│     Caretaker          │ ← 管理者，保存多个备忘录
│───────────────────────│
│ + addMemento(m)        │
│ + getMemento(index)    │
└────────────────────────┘
```

## ⚙️ 三、示例一：文本编辑器的撤销功能（最经典）

我们做一个简化版“文本编辑器”支持：

* 写文本；
* 撤销。

---

### 🧩 代码实现

```java
// 备忘录类：保存文本状态
class Memento {
    private final String content;
    public Memento(String content) { this.content = content; }
    public String getContent() { return content; }
}

// 发起者：编辑器
class Editor {
    private String content = "";

    public void type(String words) {
        content += words;
    }

    public String getContent() {
        return content;
    }

    // 创建快照
    public Memento save() {
        return new Memento(content);
    }

    // 恢复快照
    public void restore(Memento m) {
        content = m.getContent();
    }
}

// 管理者：保存历史记录
class History {
    private final Stack<Memento> history = new Stack<>();

    public void push(Memento m) { history.push(m); }
    public Memento pop() { return history.pop(); }
}

// 客户端
public class MementoDemo {
    public static void main(String[] args) {
        Editor editor = new Editor();
        History history = new History();

        editor.type("Hello ");
        history.push(editor.save());

        editor.type("World!");
        history.push(editor.save());

        editor.type(" Let's go!");
        System.out.println("当前文本: " + editor.getContent());

        // 撤销两次
        editor.restore(history.pop());
        System.out.println("撤销1次: " + editor.getContent());

        editor.restore(history.pop());
        System.out.println("撤销2次: " + editor.getContent());
    }
}
```

✅ 输出：

```
当前文本: Hello World! Let's go!
撤销1次: Hello World!
撤销2次: Hello 
```

💡 核心逻辑：

* `Editor` 是发起者；
* `Memento` 是快照；
* `History` 是管理者。

这种模式的结构恰好和浏览器历史、IDE 撤销栈一致。

## 💡 四、示例二：游戏存档 / 状态回滚

假设有一个角色对象，包含：

* 等级；
* 生命值；
* 攻击力。

玩家可以存档、读档。

---

### 🧩 实现

```java
// 备忘录
class GameMemento {
    private final int level;
    private final int hp;
    private final int attack;

    public GameMemento(int level, int hp, int attack) {
        this.level = level;
        this.hp = hp;
        this.attack = attack;
    }

    public int getLevel() { return level; }
    public int getHp() { return hp; }
    public int getAttack() { return attack; }
}

// 发起者
class GameCharacter {
    private int level;
    private int hp;
    private int attack;

    public GameCharacter(int level, int hp, int attack) {
        this.level = level; this.hp = hp; this.attack = attack;
    }

    public void fightBoss() {
        System.out.println("打 Boss！血量下降！");
        hp -= 30;
    }

    public GameMemento save() {
        return new GameMemento(level, hp, attack);
    }

    public void restore(GameMemento m) {
        this.level = m.getLevel();
        this.hp = m.getHp();
        this.attack = m.getAttack();
    }

    public void show() {
        System.out.printf("等级:%d  血量:%d  攻击:%d\n", level, hp, attack);
    }
}

// 客户端
public class GameDemo {
    public static void main(String[] args) {
        GameCharacter hero = new GameCharacter(10, 100, 50);
        hero.show();

        GameMemento save = hero.save(); // 存档

        hero.fightBoss();
        hero.show();

        System.out.println("恢复存档...");
        hero.restore(save);
        hero.show();
    }
}
```

✅ 输出：

```
等级:10  血量:100  攻击:50
打 Boss！血量下降！
等级:10  血量:70  攻击:50
恢复存档...
等级:10  血量:100  攻击:50
```

💡 这就是游戏“存档 / 读档”功能的最小原型。

## ⚙️ 五、示例三：事务回滚（轻量级思想）

数据库事务、工作流系统、甚至内存对象的操作，
都可能需要“回滚”机制。

```java
class Transaction {
    private String state;

    public void setState(String s) { state = s; }
    public String getState() { return state; }

    public Memento save() { return new Memento(state); }
    public void restore(Memento m) { state = m.getState(); }

    static class Memento {
        private final String state;
        public Memento(String s) { this.state = s; }
        public String getState() { return state; }
    }
}

public class TransactionDemo {
    public static void main(String[] args) {
        Transaction tx = new Transaction();
        tx.setState("Start");
        Transaction.Memento backup = tx.save();

        tx.setState("Modified");
        System.out.println("当前状态: " + tx.getState());

        tx.restore(backup);
        System.out.println("回滚后状态: " + tx.getState());
    }
}
```

✅ 输出：

```
当前状态: Modified
回滚后状态: Start
```

💡 这种思想在 Spring `TransactionManager` 的事务快照机制中也能看到影子。

## 🧠 六、备忘录模式的本质

> **本质就是“状态快照”机制。**

它的关键在于：

* **保存状态**（save）
* **恢复状态**（restore）
* 同时保持**封装性**（外部不能直接操作内部字段）

---

## ⚙️ 七、优缺点总结

| 优点             | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| ✅ 不破坏封装     | 对象内部状态对外保密                                         |
| ✅ 支持撤销、回滚 | 可恢复历史状态                                               |
| ✅ 可实现多级历史 | 可保存多份 Memento                                           |
| ⚠️ 缺点           | 1. 存储开销大（尤其频繁保存时）<br>2. 恢复逻辑复杂（需一致性保证） |
