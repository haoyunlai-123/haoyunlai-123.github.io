介绍下享元模式并举几个例子

非常好 👍
 你现在已经掌握了装饰者、适配器、工厂、观察者等经典模式，
 今天我们来讲另一个结构型设计模式中非常有代表性、但相对少见的 ——
 🌿 **享元模式（Flyweight Pattern）**。

------

## 🧩 一、核心思想

> **通过共享对象，来减少内存占用和对象创建的开销。**

也就是说，当系统中存在大量相似对象时，我们**将可共享的部分抽取出来，只保留必要的差异部分**。
 这样，同样的状态不会重复存储，从而显著减少资源消耗。

------

## ⚙️ 二、结构组成

| 角色                                        | 说明                           |
| ------------------------------------------- | ------------------------------ |
| **Flyweight（享元接口）**                   | 定义可共享对象的公共接口       |
| **ConcreteFlyweight（具体享元类）**         | 实现可共享的部分（内在状态）   |
| **UnsharedConcreteFlyweight（非共享对象）** | 包含独立数据（外部状态）       |
| **FlyweightFactory（享元工厂）**            | 管理共享对象的创建与复用       |
| **Client（客户端）**                        | 从工厂中获取共享对象并组合使用 |

------

## 💡 三、关键概念：内在状态 vs 外在状态

| 类型                           | 定义                         | 存储位置         |
| ------------------------------ | ---------------------------- | ---------------- |
| **内在状态 (Intrinsic State)** | 不随环境变化、可被共享的部分 | 存在享元对象内部 |
| **外在状态 (Extrinsic State)** | 会随环境变化、不能共享的部分 | 由外部传入使用   |

------

## 🏹 四、示例 1：围棋棋子（经典例子）

在围棋游戏中，棋盘上有成千上万个棋子，但其实只有两种颜色（黑/白）。
 棋子的“位置”不同，这是外部状态；而“颜色”相同，是可以共享的。

```java
// 抽象享元
interface GoPiece {
    void draw(int x, int y);
}

// 具体享元（可共享对象）
class ConcreteGoPiece implements GoPiece {
    private final String color;  // 内在状态：颜色

    public ConcreteGoPiece(String color) {
        this.color = color;
    }

    public void draw(int x, int y) {
        System.out.println(color + " 棋子放在位置 (" + x + ", " + y + ")");
    }
}

// 享元工厂
class GoPieceFactory {
    private static final Map<String, GoPiece> cache = new HashMap<>();

    public static GoPiece getPiece(String color) {
        return cache.computeIfAbsent(color, ConcreteGoPiece::new);
    }
}

// 客户端
public class Main {
    public static void main(String[] args) {
        GoPiece black1 = GoPieceFactory.getPiece("黑");
        GoPiece black2 = GoPieceFactory.getPiece("黑");
        GoPiece white = GoPieceFactory.getPiece("白");

        System.out.println(black1 == black2); // true，共享对象

        black1.draw(1, 2);
        black2.draw(3, 5);
        white.draw(2, 6);
    }
}
```

**输出：**

```
true
黑 棋子放在位置 (1, 2)
黑 棋子放在位置 (3, 5)
白 棋子放在位置 (2, 6)
```

✅ **说明：**

- “颜色”是内在状态（可共享）；
- “坐标”是外在状态（每次不同）；
- 工厂缓存确保相同颜色的对象只创建一次。

## 🪶 五、示例 2：文本编辑器中的字符对象（真实应用）

在一个文字处理器中，可能有上万字符，但每个 `'A'`、`'B'` 都可以共享同一个对象。

```java
class Character {
    private final char symbol; // 内在状态（共享）
    public Character(char symbol) { this.symbol = symbol; }

    public void display(int fontSize, String color) { // 外在状态
        System.out.println("字符: " + symbol + ", 字号: " + fontSize + ", 颜色: " + color);
    }
}

class CharacterFactory {
    private static final Map<Character, Character> pool = new HashMap<>();
    public static Character getCharacter(char c) {
        return pool.computeIfAbsent(c, Character::new);
    }
}

public class Demo {
    public static void main(String[] args) {
        Character a1 = CharacterFactory.getCharacter('A');
        Character a2 = CharacterFactory.getCharacter('A');
        Character b = CharacterFactory.getCharacter('B');

        System.out.println(a1 == a2); // true

        a1.display(12, "黑色");
        b.display(14, "蓝色");
    }
}
```

## 🧰 六、示例 3：数据库连接池 / 线程池（工程应用）

连接池就是享元思想的工程实践：

- 每个连接对象的配置（host、port、username）相同，可视为**内在状态**；
- 当前连接的“使用状态”（空闲/忙碌）为**外在状态**；
- 多次请求数据库时，不重新创建连接，而是**复用已有连接**。

```java
Connection conn = ConnectionPool.getConnection(); // 获取共享对象
ConnectionPool.release(conn);                     // 归还
```

👉 连接池、对象池、线程池都是享元思想的实际体现。

## 🔍 七、享元模式的 UML 结构

```
     ┌─────────────────────┐
     │     Flyweight       │
     │  + operation(state) │
     └─────────┬───────────┘
               │
 ┌─────────────┴─────────────┐
 │       ConcreteFlyweight   │
 │     (共享的内部状态)       │
 └─────────────┬─────────────┘
               │
       ┌───────┴────────┐
       │  FlyweightFactory │
       │  (池化与复用管理) │
       └──────────────────┘
```

## 🧠 八、优缺点总结

**✅ 优点**

- 显著减少内存消耗；
- 共享资源提升性能；
- 逻辑清晰、易扩展。

**❌ 缺点**

- 区分“内在/外在状态”需要额外复杂性；
- 管理外部状态增加代码复杂度；
- 不适合对象之间差异巨大的场景。

------

## 🚀 九、实际应用场景

| 场景             | 示例                                              |
| ---------------- | ------------------------------------------------- |
| **对象池化**     | 数据库连接池、线程池、String 常量池               |
| **图形渲染系统** | 游戏引擎中复用相同纹理、模型                      |
| **文档编辑器**   | 字体、字符对象复用                                |
| **缓存系统**     | Flyweight 对象在内存中复用（如 MyBatis 一级缓存） |

------

## 🧾 十、一句话记忆

> **享元模式：共享相同的部分，节省内存开销。**
