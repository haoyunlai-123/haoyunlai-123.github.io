你现在进入行为型模式中的另一个重量级选手 ——
👉 **命令模式（Command Pattern）**。

这个模式的出现，**彻底解耦了“请求的发起者”和“执行者”**，
广泛应用于 GUI、事务回滚、任务队列、操作日志、宏命令 等场景。

---

## 🧩 一、定义与核心思想

**命令模式（Command Pattern）**
将一个请求封装为一个对象，从而使你可以：

* 用不同的请求、队列、日志来参数化其他对象；
* 支持请求的撤销（undo）与恢复（redo）；
* 支持宏命令（组合多个命令一起执行）。

> 换句话说：
> “把一段要执行的操作封装成独立对象，
> 这样就能灵活地存储、传递、撤销或排队执行它。”

---

### 🧠 一句话理解

> “把行为（方法调用）变成对象（命令），
> 从而让调用方与执行方彻底解耦。”

---

## 🧱 二、结构图

```
           ┌────────────────────┐
           │    Client           │ ← 创建命令对象并设定接收者
           └────────┬───────────┘
                    │
           ┌────────▼────────┐
           │   Invoker       │ ← 调用者，执行命令（如按钮/菜单）
           └────────┬────────┘
                    │
           ┌────────▼────────┐
           │   Command       │ ← 命令接口（抽象行为）
           └────────┬────────┘
                    │
         ┌──────────▼──────────┐
         │  ConcreteCommand     │ ← 具体命令，封装接收者动作
         └──────────┬──────────┘
                    │
          ┌─────────▼─────────┐
          │   Receiver         │ ← 实际执行操作的对象
          └────────────────────┘
```

## ⚙️ 三、示例一：遥控器控制家电（经典入门例子）

```java
// 命令接口
interface Command {
    void execute();
}

// 接收者：灯
class Light {
    public void turnOn() {
        System.out.println("💡 灯亮了");
    }
    public void turnOff() {
        System.out.println("💡 灯灭了");
    }
}

// 具体命令类
class LightOnCommand implements Command {
    private final Light light;
    public LightOnCommand(Light light) { this.light = light; }
    public void execute() { light.turnOn(); }
}

class LightOffCommand implements Command {
    private final Light light;
    public LightOffCommand(Light light) { this.light = light; }
    public void execute() { light.turnOff(); }
}

// 调用者（Invoker）
class RemoteControl {
    private Command command;
    public void setCommand(Command command) { this.command = command; }
    public void pressButton() { command.execute(); }
}

// 客户端
public class CommandDemo {
    public static void main(String[] args) {
        Light light = new Light();
        Command on = new LightOnCommand(light);
        Command off = new LightOffCommand(light);

        RemoteControl remote = new RemoteControl();

        remote.setCommand(on);
        remote.pressButton();

        remote.setCommand(off);
        remote.pressButton();
    }
}
```

✅ 输出：

```
💡 灯亮了
💡 灯灭了
```

📘

* `Light` 是接收者（真正执行操作的对象）；
* `LightOnCommand` 是命令（封装动作）；
* `RemoteControl` 是调用者（只知道按按钮，不关心内部细节）。

## 💡 四、示例二：支持撤销（Undo）的文本编辑器命令

命令模式非常适合实现撤销/恢复功能。
每个命令执行前保存状态，执行后可以回滚。

```java
// 命令接口
interface Command {
    void execute();
    void undo();
}

// 接收者：文本编辑器
class TextEditor {
    private StringBuilder text = new StringBuilder();

    public void append(String str) {
        text.append(str);
    }

    public void deleteLast(int len) {
        text.delete(text.length() - len, text.length());
    }

    public String getText() { return text.toString(); }
}

// 具体命令：添加文本
class AddTextCommand implements Command {
    private TextEditor editor;
    private String content;

    public AddTextCommand(TextEditor editor, String content) {
        this.editor = editor;
        this.content = content;
    }

    public void execute() {
        editor.append(content);
    }

    public void undo() {
        editor.deleteLast(content.length());
    }
}

// 调用者：命令管理器
class CommandInvoker {
    private Stack<Command> history = new Stack<>();

    public void execute(Command cmd) {
        cmd.execute();
        history.push(cmd);
    }

    public void undo() {
        if (!history.isEmpty()) {
            history.pop().undo();
        }
    }
}

// 客户端
public class TextCommandDemo {
    public static void main(String[] args) {
        TextEditor editor = new TextEditor();
        CommandInvoker invoker = new CommandInvoker();

        invoker.execute(new AddTextCommand(editor, "Hello "));
        invoker.execute(new AddTextCommand(editor, "World!"));
        System.out.println("文本内容：" + editor.getText());

        invoker.undo();
        System.out.println("撤销后：" + editor.getText());
    }
}
```

✅ 输出：

```
文本内容：Hello World!
撤销后：Hello 
```

💡 命令模式在这里用栈保存历史，实现轻量级撤销功能。
这正是 IDE、Photoshop、Word 的底层原理。

## ⚙️ 五、示例三：宏命令（批量执行）

命令模式可以组合多个命令执行成“宏命令”。

```java
class MacroCommand implements Command {
    private List<Command> commands = new ArrayList<>();
    public void add(Command c) { commands.add(c); }

    public void execute() {
        for (Command c : commands) c.execute();
    }

    public void undo() {
        for (int i = commands.size() - 1; i >= 0; i--) {
            commands.get(i).undo();
        }
    }
}
```

应用：

```java
MacroCommand morningRoutine = new MacroCommand();
morningRoutine.add(new LightOnCommand(light));
morningRoutine.add(new MusicPlayCommand(musicPlayer));
morningRoutine.add(new CoffeeMachineOnCommand(coffeeMaker));

morningRoutine.execute(); // 一键启动早晨模式
```

💡 这就是“智能家居”常见的“一键执行多个动作”的实现思路。

## 🧠 六、命令模式的本质

> “用对象封装行为（请求）。”

它把“做什么”与“谁去做”彻底分开。
这样可以：

* 延迟执行；
* 存储命令（持久化、日志）；
* 撤销 / 重做；
* 组合命令。

## 🧩 七、在框架中的应用

| 场景                                   | 应用                               |
| -------------------------------------- | ---------------------------------- |
| **Java Swing**                         | `ActionListener`（按钮点击即命令） |
| **事务系统**                           | 操作封装成命令，可回滚             |
| **任务队列 / MQ**                      | 命令对象序列化发送                 |
| **数据库迁移工具（Liquibase/Flyway）** | 每个变更步骤是命令                 |
| **游戏引擎**                           | 玩家操作命令化，支持回放与撤销     |
| **Spring Batch / Quartz**              | 每个任务封装为 Command/Job         |

## ⚙️ 八、优缺点总结

| 优点                     | 说明                             |
| ------------------------ | -------------------------------- |
| ✅ 解耦请求者与执行者     | 调用方不需要知道谁执行任务       |
| ✅ 易于扩展               | 新增命令类不影响现有代码         |
| ✅ 可支持撤销、队列、日志 | 操作对象化后易于控制             |
| ⚠️ 缺点                   | 命令类可能过多（每种操作一个类） |

---

## ✅ 九、一句话总结

> 🧩 **命令模式（Command Pattern）**：
> 把请求封装成对象，让你能用同一接口排队执行、撤销、组合或延迟这些请求。
>
> 🧠 “把方法调用封装成对象，
> 让请求、排队、撤销、宏执行都变得可能。”
