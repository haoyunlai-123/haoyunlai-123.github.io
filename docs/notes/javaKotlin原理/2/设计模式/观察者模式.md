介绍下观察者模式并举几个例子

非常好 👍
 **观察者模式（Observer Pattern）** 是软件设计中非常经典的一种 **行为型设计模式**。
 它被广泛用于事件监听、消息订阅、UI 更新、通知系统等场景中。

------

## 🧩 一、核心思想

> 当一个对象的状态发生变化时，**自动通知所有依赖它的对象（观察者）**，让它们自动更新。

👉 简单说，就是实现一种“**发布-订阅（Publish-Subscribe）**”机制。

------

## ⚙️ 二、角色组成

| 角色                               | 说明                                           |
| ---------------------------------- | ---------------------------------------------- |
| **Subject（主题 / 被观察者）**     | 负责维护一组观察者列表，并在状态变化时通知它们 |
| **Observer（观察者）**             | 定义一个更新接口，当主题变化时被调用           |
| **ConcreteSubject（具体主题）**    | 保存数据状态并触发通知                         |
| **ConcreteObserver（具体观察者）** | 收到通知后做出相应更新                         |

------

## 💡 三、一个简单例子（天气预报系统）

我们有一个天气中心（主题），多个设备（观察者）订阅天气更新。

```java
// 观察者接口
interface Observer {
    void update(float temperature, float humidity);
}

// 主题接口
interface Subject {
    void registerObserver(Observer o);
    void removeObserver(Observer o);
    void notifyObservers();
}

// 具体主题（被观察者）
class WeatherStation implements Subject {
    private List<Observer> observers = new ArrayList<>();
    private float temperature;
    private float humidity;

    public void setMeasurements(float temperature, float humidity) {
        this.temperature = temperature;
        this.humidity = humidity;
        notifyObservers();  // 状态变化时通知
    }

    public void registerObserver(Observer o) { observers.add(o); }
    public void removeObserver(Observer o) { observers.remove(o); }
    public void notifyObservers() {
        for (Observer o : observers) {
            o.update(temperature, humidity);
        }
    }
}

// 具体观察者
class PhoneDisplay implements Observer {
    public void update(float temperature, float humidity) {
        System.out.println("📱 手机显示：温度=" + temperature + ", 湿度=" + humidity);
    }
}

class TvDisplay implements Observer {
    public void update(float temperature, float humidity) {
        System.out.println("📺 电视显示：温度=" + temperature + ", 湿度=" + humidity);
    }
}

// 客户端
public class Main {
    public static void main(String[] args) {
        WeatherStation station = new WeatherStation();
        station.registerObserver(new PhoneDisplay());
        station.registerObserver(new TvDisplay());

        station.setMeasurements(25.5f, 60.0f);
        station.setMeasurements(28.3f, 55.5f);
    }
}
```

**输出：**

```
📱 手机显示：温度=25.5, 湿度=60.0
📺 电视显示：温度=25.5, 湿度=60.0
📱 手机显示：温度=28.3, 湿度=55.5
📺 电视显示：温度=28.3, 湿度=55.5
```

👉 当天气数据更新时，所有注册的显示设备都会收到通知。

## 🧰 四、Java 自带实现（内置观察者机制）

Java 早期提供了内置的 `java.util.Observable` 与 `java.util.Observer`（已废弃，但原理仍可学）。

```java
class WeatherData extends Observable {
    private float temp;
    void setTemp(float t) {
        this.temp = t;
        setChanged();      // 标记状态变化
        notifyObservers(t); // 通知所有观察者
    }
}

class Display implements Observer {
    public void update(Observable o, Object arg) {
        System.out.println("温度更新：" + arg);
    }
}

public class Demo {
    public static void main(String[] args) {
        WeatherData data = new WeatherData();
        data.addObserver(new Display());
        data.setTemp(26.3f);
    }
}
```

## 🧠 五、现实中的应用场景

| 场景                   | 示例                                                   |
| ---------------------- | ------------------------------------------------------ |
| **GUI 事件监听**       | 按钮点击事件触发监听器（Swing、JavaFX、Android）       |
| **消息通知系统**       | 邮件订阅、微信推送、事件广播                           |
| **数据库变更监听**     | 触发器、ORM 实体监听（如 Hibernate 的 EntityListener） |
| **分布式事件总线**     | Kafka / RabbitMQ 的 pub-sub 机制                       |
| **前端框架响应式更新** | Vue 的数据劫持、React 的状态订阅机制                   |

## ⚡ 六、Kotlin 实现（更简洁）

Kotlin 中可以用 **函数式回调** 实现观察者机制，非常自然：

```kotlin
class WeatherStation {
    private val listeners = mutableListOf<(Float) -> Unit>()
    fun addListener(listener: (Float) -> Unit) = listeners.add(listener)
    fun setTemp(temp: Float) = listeners.forEach { it(temp) }
}

fun main() {
    val station = WeatherStation()
    station.addListener { println("手机显示温度: $it") }
    station.addListener { println("网页显示温度: $it") }
    station.setTemp(25.5f)
}
```

**输出：**

```
手机显示温度: 25.5
网页显示温度: 25.5
```

👉 利用 Lambda 函数实现观察者，代码极简。

## 🔍 七、优缺点总结

**✅ 优点**

- 实现对象间的“解耦”；
- 支持一对多自动通知；
- 容易扩展新的观察者。

**❌ 缺点**

- 通知链过长时，调试困难；
- 不保证通知顺序；
- 如果被观察者频繁变化，性能会受影响。

------

## 🧾 八、一句话记忆

> **观察者模式 = 一对多 + 自动通知 + 解耦。**
