介绍下创建者模式并举几个例子

非常好 👍！你现在走到设计模式的最后一个创建型模式了 ——
 **创建者模式（Builder Pattern）**。

它在实际开发里**出场率极高**（尤其在 Java），
 几乎所有复杂对象的构造（如 `StringBuilder`、`Stream.Builder`、`HttpRequest.Builder`）
 都在用它。

------

## 🧩 一、概念简介

**创建者模式（Builder Pattern）**
 是一种用于**分步骤创建复杂对象**的设计模式。

> 它将对象的构建（build）与表示（representation）分离，
>  使得相同的构建过程可以创建不同的表示形式。

------

### 🧠 换成人话：

当一个对象的构造函数有很多参数、或者需要复杂组合时，
 与其写成：

```java
new Computer("Intel i9", "32GB", "1TB", "RTX4090", true, false, true)
```

不如用 Builder 分步骤构建：

```java
Computer pc = new Computer.Builder()
    .cpu("Intel i9")
    .memory("32GB")
    .gpu("RTX4090")
    .hasWifi(true)
    .build();
```

可读性、可维护性、可扩展性都提升一个档次。

## ⚙️ 三、代码示例一：最常见的 Java Builder（链式调用）

```java
// 产品类
public class Computer {
    private final String cpu;
    private final String memory;
    private final String storage;
    private final String gpu;
    private final boolean hasWifi;
    private final boolean hasBluetooth;

    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.memory = builder.memory;
        this.storage = builder.storage;
        this.gpu = builder.gpu;
        this.hasWifi = builder.hasWifi;
        this.hasBluetooth = builder.hasBluetooth;
    }

    // Builder 静态内部类
    public static class Builder {
        private String cpu;
        private String memory;
        private String storage;
        private String gpu;
        private boolean hasWifi;
        private boolean hasBluetooth;

        public Builder cpu(String cpu) { this.cpu = cpu; return this; }
        public Builder memory(String memory) { this.memory = memory; return this; }
        public Builder storage(String storage) { this.storage = storage; return this; }
        public Builder gpu(String gpu) { this.gpu = gpu; return this; }
        public Builder hasWifi(boolean val) { this.hasWifi = val; return this; }
        public Builder hasBluetooth(boolean val) { this.hasBluetooth = val; return this; }

        public Computer build() {
            return new Computer(this);
        }
    }

    @Override
    public String toString() {
        return String.format("Computer[%s, %s, %s, %s, wifi=%s, bt=%s]",
                cpu, memory, storage, gpu, hasWifi, hasBluetooth);
    }
}

// 客户端
public class Main {
    public static void main(String[] args) {
        Computer pc = new Computer.Builder()
                .cpu("Intel i9")
                .memory("32GB")
                .storage("1TB SSD")
                .gpu("RTX4090")
                .hasWifi(true)
                .hasBluetooth(true)
                .build();

        System.out.println(pc);
    }
}
```

✅ 输出：

```
Computer[Intel i9, 32GB, 1TB SSD, RTX4090, wifi=true, bt=true]
```

## 💡 四、示例二：带“Director”（指挥者）的标准版

有时你希望封装“构建流程”，例如：

> “如何组装一台游戏电脑 vs 办公电脑”。

```java
// 产品
class Computer {
    String cpu, gpu, ram, storage;
    public String toString() { return cpu + "/" + gpu + "/" + ram + "/" + storage; }
}

// Builder 抽象接口
interface Builder {
    void buildCPU();
    void buildGPU();
    void buildRAM();
    void buildStorage();
    Computer getResult();
}

// 具体建造者
class GamingComputerBuilder implements Builder {
    Computer pc = new Computer();
    public void buildCPU() { pc.cpu = "i9"; }
    public void buildGPU() { pc.gpu = "RTX4090"; }
    public void buildRAM() { pc.ram = "32GB"; }
    public void buildStorage() { pc.storage = "1TB SSD"; }
    public Computer getResult() { return pc; }
}

// 指挥者
class Director {
    public void construct(Builder builder) {
        builder.buildCPU();
        builder.buildGPU();
        builder.buildRAM();
        builder.buildStorage();
    }
}

// 客户端
public class Client {
    public static void main(String[] args) {
        Director director = new Director();
        Builder builder = new GamingComputerBuilder();
        director.construct(builder);
        Computer pc = builder.getResult();
        System.out.println(pc);
    }
}
```

✅ 优点：

- `Director` 控制构建步骤；
- `Builder` 控制细节；
- 不同 `Builder` 可以创建不同类型产品。

## 🧬 五、示例三：链式 JSON 构造（真实应用）

很多库内部都用 Builder，比如：

```java
String json = new JsonBuilder()
        .beginObject()
        .add("name", "Alice")
        .add("age", 25)
        .endObject()
        .build();
```

自定义实现（简化版）：

```java
class JsonBuilder {
    private final StringBuilder sb = new StringBuilder();

    public JsonBuilder beginObject() {
        sb.append("{");
        return this;
    }
    public JsonBuilder add(String key, Object value) {
        if (sb.charAt(sb.length() - 1) != '{') sb.append(",");
        sb.append("\"").append(key).append("\":\"").append(value).append("\"");
        return this;
    }
    public JsonBuilder endObject() {
        sb.append("}");
        return this;
    }
    public String build() {
        return sb.toString();
    }
}
```

输出：

```
{"name":"Alice","age":"25"}
```

## 🧩 六、与工厂模式的区别

| 特点     | 工厂模式                 | 创建者模式                                      |
| -------- | ------------------------ | ----------------------------------------------- |
| 目标     | 直接创建完整对象         | 分步骤构建复杂对象                              |
| 参数     | 传入必要参数             | 支持链式可选参数                                |
| 封装层次 | 一次性 new               | 可拆解构建过程                                  |
| 典型应用 | 简单对象                 | 复杂组合对象                                    |
| 例子     | `CarFactory.createCar()` | `Car.Builder().engine(...).wheels(...).build()` |

------

## ✅ 七、总结一句话

> **创建者模式（Builder Pattern）**：
>  通过**分步骤构建对象**来代替冗长的构造函数调用。
>  它让对象创建更灵活、更可读，并支持复杂组合的扩展。
