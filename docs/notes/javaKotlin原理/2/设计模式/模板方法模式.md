介绍下行为型模式中的模板方法

非常好 👍！
你现在进入到 **行为型设计模式（Behavioral Patterns）** 的部分了。
这些模式的重点不在结构，而在**对象之间的行为与流程控制**。

我们今天讲的 ——
👉 **模板方法模式（Template Method Pattern）**，
是行为型模式中最基础、最常考、最常用的一个。

---

## 🧩 一、定义与核心思想

**模板方法模式（Template Method Pattern）**
定义了一个操作中算法的**框架（模板）**，
并将**某些步骤延迟到子类中**实现。

> 换句话说：
> 父类定义算法的“固定流程骨架”，
> 子类只需覆盖其中的可变步骤，而无需改动整体流程。

---

### 🧠 一句话理解

> “父类定套路，子类填细节。”

---

## 🧱 二、结构图

```
           ┌────────────────────┐
           │  AbstractClass     │ ← 抽象父类
           │────────────────────│
           │ + templateMethod() │ ← 定义算法骨架
           │ + step1()          │ ← 固定步骤
           │ + step2()          │ ← 抽象步骤（交给子类）
           └────────┬───────────┘
                    │
        ┌───────────┴───────────┐
        │                       │
┌───────▼────────┐     ┌────────▼───────┐
│ ConcreteClassA  │     │ ConcreteClassB │
│────────────────│     │────────────────│
│ + step2()实现A │     │ + step2()实现B │
└────────────────┘     └────────────────┘
```

## ⚙️ 三、示例一：制作饮料（经典例子）

冲饮料都有固定步骤：

1. 烧水
2. 冲泡
3. 倒入杯中
4. 加调料（可变）

不同饮料只在“冲泡”和“加料”步骤上不同。
于是我们用模板方法封装整个流程。

```java
abstract class Beverage {
    // 模板方法（final防止子类改流程）
    public final void prepareRecipe() {
        boilWater();
        brew();
        pourInCup();
        addCondiments();
    }

    private void boilWater() {
        System.out.println("烧开水");
    }

    private void pourInCup() {
        System.out.println("倒入杯中");
    }

    // 抽象方法，交由子类实现
    protected abstract void brew();
    protected abstract void addCondiments();
}

// 子类1：咖啡
class Coffee extends Beverage {
    protected void brew() {
        System.out.println("冲泡咖啡");
    }
    protected void addCondiments() {
        System.out.println("加糖和牛奶");
    }
}

// 子类2：茶
class Tea extends Beverage {
    protected void brew() {
        System.out.println("浸泡茶叶");
    }
    protected void addCondiments() {
        System.out.println("加柠檬");
    }
}

// 客户端
public class TemplateDemo {
    public static void main(String[] args) {
        System.out.println("制作咖啡：");
        new Coffee().prepareRecipe();

        System.out.println("\n制作茶：");
        new Tea().prepareRecipe();
    }
}
```

✅ 输出：

```
制作咖啡：
烧开水
冲泡咖啡
倒入杯中
加糖和牛奶

制作茶：
烧开水
浸泡茶叶
倒入杯中
加柠檬
```

💡 “烧水 → 倒杯”是固定模板；
“冲泡 → 加料”由子类自由实现。

## 🧠 四、示例二：Spring 框架中的模板方法思想

Spring 中有非常多类都用到了模板方法模式，比如：

| 类名                         | 模板方法                    | 可重写步骤                |
| ---------------------------- | --------------------------- | ------------------------- |
| `JdbcTemplate`               | `execute()`                 | 回调中自定义 SQL 执行逻辑 |
| `AbstractApplicationContext` | `refresh()`                 | 子类定义初始化细节        |
| `AbstractHandlerMapping`     | `getHandler()`              | 子类定义请求匹配规则      |
| `AbstractView`               | `renderMergedOutputModel()` | 子类实现具体视图渲染      |

例如 Spring 的 `JdbcTemplate`：

```java
jdbcTemplate.query("SELECT * FROM user WHERE id = ?", rs -> {
    System.out.println(rs.getString("name"));
}, 1);
```

`JdbcTemplate` 定义了模板：

* 建立连接
* 执行 SQL
* 关闭连接
* 异常处理

用户只需传入一个小回调（实现部分逻辑），
整个流程自动执行。
这就是 **模板方法模式 + 回调机制** 的经典结合。

## ⚙️ 五、示例三：算法框架模板

假设我们要实现一个“数据处理框架”，包含：

1. 加载数据
2. 清洗
3. 分析
4. 输出结果

不同数据源（CSV / Database）实现方式不同。

```java
abstract class DataProcessor {
    public final void process() {
        loadData();
        cleanData();
        analyzeData();
        exportResult();
    }

    protected abstract void loadData();
    protected void cleanData() {
        System.out.println("默认清洗逻辑");
    }
    protected abstract void analyzeData();
    protected abstract void exportResult();
}

class CSVProcessor extends DataProcessor {
    protected void loadData() { System.out.println("读取CSV文件"); }
    protected void analyzeData() { System.out.println("分析CSV数据"); }
    protected void exportResult() { System.out.println("输出结果为report.csv"); }
}

class DBProcessor extends DataProcessor {
    protected void loadData() { System.out.println("连接数据库并加载数据"); }
    protected void analyzeData() { System.out.println("执行SQL分析"); }
    protected void exportResult() { System.out.println("输出到数据库表"); }
}

public class DataTemplateDemo {
    public static void main(String[] args) {
        new CSVProcessor().process();
        System.out.println("------");
        new DBProcessor().process();
    }
}
```

✅ 输出：

```
读取CSV文件
默认清洗逻辑
分析CSV数据
输出结果为report.csv
------
连接数据库并加载数据
默认清洗逻辑
执行SQL分析
输出到数据库表
```

💡 模板固定了算法骨架，不同数据源实现细节不同。

## 🧬 六、模板方法模式的关键点

| 概念                             | 含义                                                        |
| -------------------------------- | ----------------------------------------------------------- |
| **模板方法（templateMethod）**   | 定义整个算法执行流程的骨架                                  |
| **具体步骤（concrete methods）** | 父类中已实现的固定部分                                      |
| **抽象步骤（abstract methods）** | 留给子类实现的可变部分                                      |
| **钩子方法（hook methods）**     | 子类可选择性重写，用于控制流程（例如：`isAddCondiments()`） |

---

### 🔧 钩子方法示例

让子类可控制是否执行某一步：

```java
abstract class Beverage {
    public final void prepareRecipe() {
        boilWater();
        brew();
        pourInCup();
        if (customerWantsCondiments()) {
            addCondiments();
        }
    }
    abstract void brew();
    abstract void addCondiments();
    boolean customerWantsCondiments() { return true; } // hook
    void boilWater() { System.out.println("烧水"); }
    void pourInCup() { System.out.println("倒杯"); }
}

class Coffee extends Beverage {
    void brew() { System.out.println("冲咖啡"); }
    void addCondiments() { System.out.println("加糖加奶"); }
    boolean customerWantsCondiments() {
        return false; // 用户不想加
    }
}
```

## 🧩 七、优缺点总结

| 优点           | 说明                                               |
| -------------- | -------------------------------------------------- |
| ✅ 封装不变部分 | 父类固定通用流程                                   |
| ✅ 扩展灵活     | 子类仅需重写差异步骤                               |
| ✅ 复用性高     | 统一模板减少重复代码                               |
| ⚠️ 缺点         | 子类过多、依赖继承、灵活性受限（无法动态改变流程） |

| 场景                   | 示例                                   |
| ---------------------- | -------------------------------------- |
| 算法框架固定但细节不同 | 各种数据处理流程（如 ETL）             |
| 多子类共享逻辑         | 抽象 Controller / Service 层逻辑       |
| 框架设计               | Spring、JUnit、Servlet、MyBatis 都用到 |

---

## ✅ 九、一句话总结

> 🧩 **模板方法模式（Template Method Pattern）**：
> 定义算法的固定执行流程，把变化的部分交给子类实现。
> 它是 “**继承 + 控制反转（Inversion of Control）**” 的经典体现。
>
> 🔹 父类规定套路，
> 🔹 子类只需填空。
