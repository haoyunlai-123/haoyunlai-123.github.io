你现在已经学习到行为型模式的最后一个经典成员 ——
👉 **解释器模式（Interpreter Pattern）**。

它是所有**DSL（领域特定语言）**、**表达式解析器**、**规则引擎**、**公式计算系统**、甚至**正则表达式实现**的理论基础。

虽然它不是最常用的模式，但却是最“有思想含量”的之一。
理解它，你就能看懂“如何从文本变成可执行逻辑”这类系统（例如：SQL、EL 表达式、正则、脚本引擎）。

---

## 🧩 一、定义与核心思想

**解释器模式（Interpreter Pattern）**
给定一种语言，定义它的语法表示，并定义一个解释器，
这个解释器根据该表示来解释语言中的句子。

> 换句话说：
> “把一段表达式结构化，并能用程序执行（解释）它。”

---

### 🧠 一句话理解

> “让代码能像读语言一样被解释执行。”

---

## 🧱 二、结构图

```
┌────────────────────┐
│ AbstractExpression │ ← 抽象表达式
│ + interpret(ctx)   │
└────────┬───────────┘
         │
 ┌───────▼────────────┐
 │ TerminalExpression │ ← 终结符表达式（数字、变量等）
 └───────┬────────────┘
         │
 ┌───────▼────────────┐
 │ NonTerminalExpr.   │ ← 非终结符表达式（加减乘除等）
 └────────────────────┘
 
┌────────────────────┐
│    Context          │ ← 上下文（环境变量）
└────────────────────┘
```

## ⚙️ 三、示例一：四则运算解释器（经典例子）

我们实现一个简单的表达式解释器，比如：

```
10 + 5 - 3
```

---

### 🧩 代码实现

```java
// 抽象表达式
interface Expression {
    int interpret();
}

// 终结符表达式（数字）
class NumberExpression implements Expression {
    private int value;
    public NumberExpression(int value) { this.value = value; }
    public int interpret() { return value; }
}

// 非终结符表达式（加法）
class AddExpression implements Expression {
    private Expression left, right;
    public AddExpression(Expression left, Expression right) {
        this.left = left; this.right = right;
    }
    public int interpret() {
        return left.interpret() + right.interpret();
    }
}

// 非终结符表达式（减法）
class SubtractExpression implements Expression {
    private Expression left, right;
    public SubtractExpression(Expression left, Expression right) {
        this.left = left; this.right = right;
    }
    public int interpret() {
        return left.interpret() - right.interpret();
    }
}

// 客户端：构造语法树并解释
public class InterpreterDemo {
    public static void main(String[] args) {
        // 表达式: 10 + 5 - 3
        Expression expr = new SubtractExpression(
            new AddExpression(
                new NumberExpression(10),
                new NumberExpression(5)
            ),
            new NumberExpression(3)
        );

        System.out.println("解释结果: " + expr.interpret());
    }
}
```

✅ 输出：

```
解释结果: 12
```

💡 本质上，我们是在“手动构造”一个抽象语法树（AST），
`interpret()` 方法即为解释执行。

## 💡 四、示例二：变量表达式 + 环境上下文

支持表达式：

```
a + b - c
```

上下文：

```
a = 10, b = 5, c = 2
```

---

### 🧩 代码实现

```java
import java.util.*;

// 上下文：存储变量
class Context {
    private Map<String, Integer> vars = new HashMap<>();
    public void assign(String var, int value) {
        vars.put(var, value);
    }
    public int lookup(String var) {
        return vars.get(var);
    }
}

// 抽象表达式
interface Expression {
    int interpret(Context ctx);
}

// 终结符：变量
class VarExpression implements Expression {
    private String name;
    public VarExpression(String name) { this.name = name; }
    public int interpret(Context ctx) {
        return ctx.lookup(name);
    }
}

// 非终结符：加法
class AddExpression implements Expression {
    private Expression left, right;
    public AddExpression(Expression l, Expression r) {
        this.left = l; this.right = r;
    }
    public int interpret(Context ctx) {
        return left.interpret(ctx) + right.interpret(ctx);
    }
}

// 非终结符：减法
class SubExpression implements Expression {
    private Expression left, right;
    public SubExpression(Expression l, Expression r) {
        this.left = l; this.right = r;
    }
    public int interpret(Context ctx) {
        return left.interpret(ctx) - right.interpret(ctx);
    }
}

// 客户端
public class InterpreterWithVars {
    public static void main(String[] args) {
        Context ctx = new Context();
        ctx.assign("a", 10);
        ctx.assign("b", 5);
        ctx.assign("c", 2);

        // 表达式: a + b - c
        Expression expr = new SubExpression(
            new AddExpression(
                new VarExpression("a"),
                new VarExpression("b")
            ),
            new VarExpression("c")
        );

        System.out.println("结果：" + expr.interpret(ctx)); // 13
    }
}
```

✅ 输出：

```
结果：13
```

💡
这就像一个“迷你解释器”，
`Context` 提供“环境变量”，`Expression` 提供“语法规则”。

## ⚙️ 五、示例三：布尔表达式解释器（DSL 实例）

表达式：

```
x AND y OR z
```

上下文：

```
x=true, y=false, z=true
```

---

### 🧩 实现

```java
interface BoolExpr {
    boolean interpret(Map<String, Boolean> context);
}

class VarExpr implements BoolExpr {
    private String name;
    public VarExpr(String name) { this.name = name; }
    public boolean interpret(Map<String, Boolean> context) {
        return context.get(name);
    }
}

class AndExpr implements BoolExpr {
    private BoolExpr left, right;
    public AndExpr(BoolExpr l, BoolExpr r) { left = l; right = r; }
    public boolean interpret(Map<String, Boolean> ctx) {
        return left.interpret(ctx) && right.interpret(ctx);
    }
}

class OrExpr implements BoolExpr {
    private BoolExpr left, right;
    public OrExpr(BoolExpr l, BoolExpr r) { left = l; right = r; }
    public boolean interpret(Map<String, Boolean> ctx) {
        return left.interpret(ctx) || right.interpret(ctx);
    }
}

public class BoolInterpreterDemo {
    public static void main(String[] args) {
        Map<String, Boolean> ctx = new HashMap<>();
        ctx.put("x", true);
        ctx.put("y", false);
        ctx.put("z", true);

        BoolExpr expr = new OrExpr(
            new AndExpr(new VarExpr("x"), new VarExpr("y")),
            new VarExpr("z")
        );

        System.out.println("结果：" + expr.interpret(ctx));
    }
}
```

✅ 输出：

```
结果：true
```

💡
这就相当于解释逻辑表达式的执行过程。
在规则引擎、工作流系统、监控告警条件判断中都非常常见。

## 🧠 六、解释器模式的本质

> **解释器模式的本质就是“用面向对象的方式描述语法树，并实现其求值逻辑”。**

即：

* 语法规则 → 类结构
* 表达式节点 → 对象
* 解释逻辑 → `interpret()` 方法

最终形成一个“小型编程语言”内部解释系统。

## ⚙️ 七、优缺点总结

| 优点         | 说明                                    |
| ---------- | ------------------------------------- |
| ✅ 易于扩展     | 新增语法符号只需新增类                           |
| ✅ 直观表达语言语法 | 结构与文法一一对应                             |
| ✅ 解耦语法与执行  | 语义逻辑集中在解释器类                           |
| ⚠️ 缺点      | 1. 类数量激增<br>2. 复杂文法难维护<br>3. 性能低于编译执行 |

## 💡 八、典型应用场景

| 场景          | 应用                    |
| ----------- | --------------------- |
| 编译器 / 脚本解释器 | 解释表达式、AST             |
| 规则引擎        | `if a > 10 and b < 5` |
| 表达式计算       | EL、SpEL、OGNL          |
| SQL / 正则解析器 | 内部表达式语义解释             |
| 机器人指令解析     | “move 10 steps left”  |
| 模板引擎        | 变量解析、表达式求值            |

## ✅ 十、一句话总结

> 🧩 **解释器模式（Interpreter Pattern）**：
> 用类来表示语言的语法规则，用对象结构来解释执行表达式。

> 🧠 “它让你可以自己造一个迷你语言解释器。”
