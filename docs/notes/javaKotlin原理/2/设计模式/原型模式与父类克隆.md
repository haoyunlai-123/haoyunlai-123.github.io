介绍下原型模式并举几个例子

非常好 👍 —— 你现在已经学了工厂、代理、装饰、单例这些经典模式，
 接下来我们讲的 **原型模式（Prototype Pattern）** 是创建型设计模式中最灵活、但最容易被忽视的一个。

------

## 🧩 一、概念简介

**原型模式（Prototype Pattern）**
 是一种通过「复制（克隆）」现有对象来创建新对象的设计模式。

👉 核心思想：

> 不通过 `new` 关键字去创建对象，而是用一个已经存在的对象作为原型，通过复制（clone）得到新的实例。

这种模式常用于：

- 创建对象代价很高（比如复杂初始化）
- 或需要在运行时动态生成大量相似对象的场景。

------

## 🧠 二、定义（来自 GoF）

> **Prototype** 模式：
>  使用原型实例指定创建对象的种类，并通过拷贝这些原型创建新的对象。

------

## 🧱 三、类图结构

```
          ┌──────────────┐
          │ Prototype    │ ← 接口，定义 clone() 方法
          └──────┬───────┘
                 │
          ┌──────▼────────┐
          │ ConcreteProto │ ← 实现 clone()，返回自身的副本
          └───────────────┘
```

## ⚙️ 四、代码示例一：最经典的 Java 版本

```java
// 1. 定义原型接口
public interface Prototype extends Cloneable {
    Prototype clone();
}

// 2. 具体实现类
public class Shape implements Prototype {
    private String type;
    private int[] points;

    public Shape(String type, int[] points) {
        this.type = type;
        this.points = points;
    }

    @Override
    public Shape clone() {
        try {
            Shape copy = (Shape) super.clone();
            copy.points = points.clone(); // 深拷贝数组
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new AssertionError();
        }
    }

    @Override
    public String toString() {
        return type + "@" + points.hashCode();
    }
}

// 3. 使用
public class Main {
    public static void main(String[] args) {
        Shape circle = new Shape("Circle", new int[]{1, 2, 3});
        Shape clone = circle.clone();

        System.out.println(circle); // Circle@123456
        System.out.println(clone);  // Circle@654321 (不同对象)
    }
}
```

✅ 输出说明：

- `clone()` 创建了一个新对象；
- 新对象和原对象共享结构但独立存在。

## 💡 五、代码示例二：原型注册表（Registry）

在复杂系统中，我们会维护一个“原型仓库”，
 每种类型只保留一个模板对象，用来复制。

```java
import java.util.HashMap;
import java.util.Map;

abstract class Document implements Cloneable {
    public abstract Document clone();
}

class Resume extends Document {
    String name;
    @Override
    public Resume clone() {
        try {
            return (Resume) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}

class Certificate extends Document {
    String title;
    @Override
    public Certificate clone() {
        try {
            return (Certificate) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}

// 原型仓库
class PrototypeRegistry {
    private static final Map<String, Document> cache = new HashMap<>();

    static {
        cache.put("resume", new Resume());
        cache.put("certificate", new Certificate());
    }

    public static Document getPrototype(String type) {
        return cache.get(type).clone();
    }
}

public class Client {
    public static void main(String[] args) {
        Document d1 = PrototypeRegistry.getPrototype("resume");
        Document d2 = PrototypeRegistry.getPrototype("resume");
        System.out.println(d1 != d2); // true
    }
}
```

📦 应用场景：

- 报表模板（Word/PDF 模板）
- 游戏对象（敌人、装备）
- 数据导入导出时的模板复制

## 🧬 六、示例三：浅拷贝 vs 深拷贝

```java
class Person implements Cloneable {
    String name;
    Address address;

    @Override
    protected Person clone() throws CloneNotSupportedException {
        Person copy = (Person) super.clone();
        // 深拷贝 Address
        copy.address = address.clone();
        return copy;
    }
}

class Address implements Cloneable {
    String city;
    @Override
    protected Address clone() throws CloneNotSupportedException {
        return (Address) super.clone();
    }
}
```

🧠 要点：

- `super.clone()` 是浅拷贝；
- 想彻底独立，必须**手动深拷贝引用字段**。

>  本质上是“通过拷贝而不是 new”。

你这问题问得太到位了 👏——这是理解 Java `clone()` 机制的核心：

> ✅ 为什么 `clone()` 一般都要 `super.clone()`？
>  ✅ 它到底在干什么？
>  ✅ 在继承体系中它能不能自动处理父类逻辑？

我们来彻底讲清楚底层机制，这个问题理解透了，原型模式、对象拷贝、深浅复制、继承链的行为都能搞定。

------

## 🧩 一、`super.clone()` 的真实作用

`super.clone()` 调用的是 `Object` 类中的 **native 方法**：

```java
protected native Object clone() throws CloneNotSupportedException;
```

这个方法在 JVM 层做了一件事：

> **按位复制（bitwise copy）当前对象的所有字段到一个新的内存空间中。**

也就是所谓的 **浅拷贝（shallow copy）**。

------

### 🔍 举例说明：

假设有：

```java
class Person implements Cloneable {
    String name;
    Address addr;

    @Override
    protected Person clone() throws CloneNotSupportedException {
        return (Person) super.clone();
    }
}
```

执行 `p2 = p1.clone()` 时，`super.clone()` 会做两件事：

1. **分配一块新的内存（与 p1 大小相同）**
2. **将 p1 的所有字段（包括父类的字段）逐位拷贝到新对象中**

所以：

- `name`、`addr` 的引用被复制（浅拷贝）；
- `addr` 本身的内容不会被递归复制；
- 所以 `p1.addr == p2.addr` → `true`。

------

## 🧱 二、为什么要调 `super.clone()`（而不是自己 new）

1. **`Object.clone()` 是唯一能直接复制内存布局的方法**
   - 你 `new` 出来的对象必须自己手动复制每个字段。
   - `super.clone()` 一次性复制所有字段，包含父类字段（私有的也会复制）。
2. **`super.clone()` 能确保父类的状态被正确拷贝**
   - 你自己 new 只会初始化当前类字段；
   - `super.clone()` 会把从 `Object` 到所有父类字段的内容都完整复制。
3. **继承层级多时，它是“链式安全”的复制起点**
   - 每一层都可以在自己的 `clone()` 方法中：
     - 先调 `super.clone()`（复制上层字段）
     - 再补上自己需要的深拷贝逻辑。

------

## 🧠 三、继承场景中的调用链逻辑

假设有继承层：

```
Object
 └─ Person
      └─ Employee
```

每层都实现 `clone()`：

```java
class Person implements Cloneable {
    Address addr;
    @Override
    protected Person clone() throws CloneNotSupportedException {
        Person copy = (Person) super.clone(); // 调用 Object.clone()
        copy.addr = addr.clone();             // 深拷贝引用字段
        return copy;
    }
}

class Employee extends Person {
    Department dept;
    @Override
    protected Employee clone() throws CloneNotSupportedException {
        Employee copy = (Employee) super.clone(); // 调 Person.clone()
        copy.dept = dept.clone();
        return copy;
    }
}
```

执行：

```java
Employee e2 = e1.clone();
```

执行链如下：

1. `Employee.clone()` 调 `super.clone()` → 进入 `Person.clone()`
2. `Person.clone()` 再调 `super.clone()` → 进入 `Object.clone()`
3. `Object.clone()` 直接做 bitwise copy（整条继承链的字段）
4. 返回逐层向上，子类在返回途中补充自己的深拷贝逻辑

📘 这样就保证了：

- 所有父类字段都被安全复制；
- 每层都能定制自己的深拷贝逻辑。

------

## ⚙️ 四、如果你不调用 `super.clone()` 会怎样？

```java
@Override
protected Person clone() {
    Person p = new Person();  // ❌ 手动 new
    p.addr = this.addr.clone();
    return p;
}
```

问题：

- 父类的私有字段复制不到；
- 父类状态（如内部标志、缓存、统计）全丢；
- 一旦父类以后多了字段，你的 clone() 就不再正确；
- 继承链断裂，破坏“向上兼容”。

所以 `super.clone()` 是一种“结构安全拷贝”，
 **它让你不用担心父类内部状态复制问题。**

------

## 🧬 五、super.clone() ≠ 深拷贝

这点很关键！

- `super.clone()` 只负责“拷贝对象自身的字段值”
- 它不会递归 clone 这些字段引用的对象
- 想实现深拷贝，你必须**在 clone() 里手动递归克隆引用字段**

例：

```java
Person copy = (Person) super.clone();
copy.addr = addr.clone(); // 手动深拷贝
```

------

## 💡 六、总结：一张表理解

| 问题     | super.clone() 做什么？       | 为什么要调？         | 会不会自动深拷贝？ |
| -------- | ---------------------------- | -------------------- | ------------------ |
| 功能     | bitwise 拷贝当前对象         | 确保父类字段被复制   | ❌ 不会             |
| 内部实现 | JVM native C++ 级复制        | 防止丢失父类私有状态 | 需要手动深拷贝     |
| 调用链   | 子类 → 父类 → Object.clone() | 逐层安全复制         | 每层可扩展         |
| 如果不调 | new 出来的对象没复制父类内容 | 破坏继承链拷贝一致性 | ❌                  |

------

## ✅ 七、一句话总结

> `super.clone()` 是整个 Java 对象克隆机制的“入口钥匙”：
>  它调用 `Object.clone()` 完成底层内存级浅拷贝，
>  确保所有父类字段被安全复制，
>  而你只需要在子类里补充自己的深拷贝逻辑。
