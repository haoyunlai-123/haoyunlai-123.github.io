## 🧩 一、定义与核心思想

**策略模式（Strategy Pattern）**
定义了一系列算法（或行为），
并将每个算法封装起来，使它们可以互相替换，
而不影响使用这些算法的客户端。

> 换句话说：
> 把一堆 `if...else` 的行为逻辑封装成独立的策略类，
> 客户端通过接口来“选择”策略，而不是硬编码条件。

---

### 🧠 一句话理解

> “把算法（行为）抽象成接口，让使用者能在运行时自由切换。”

## ⚙️ 三、示例一：支付策略（常见业务场景）

假设你有三种支付方式：支付宝、微信、信用卡。
传统写法可能是：

```java
if (payType.equals("AliPay")) { ... }
else if (payType.equals("WeChat")) { ... }
else if (payType.equals("Card")) { ... }
```

→ 代码越来越长，不易扩展。

用策略模式我们可以优雅地解耦👇

```java
// 策略接口
interface PaymentStrategy {
    void pay(double amount);
}

// 具体策略类
class AliPayStrategy implements PaymentStrategy {
    public void pay(double amount) {
        System.out.println("使用支付宝支付：" + amount + " 元");
    }
}

class WeChatPayStrategy implements PaymentStrategy {
    public void pay(double amount) {
        System.out.println("使用微信支付：" + amount + " 元");
    }
}

class CreditCardStrategy implements PaymentStrategy {
    public void pay(double amount) {
        System.out.println("使用信用卡支付：" + amount + " 元");
    }
}

// 环境类
class PaymentContext {
    private PaymentStrategy strategy;

    public void setStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }

    public void pay(double amount) {
        strategy.pay(amount);
    }
}

// 客户端
public class StrategyDemo {
    public static void main(String[] args) {
        PaymentContext context = new PaymentContext();

        context.setStrategy(new AliPayStrategy());
        context.pay(100.0);

        context.setStrategy(new WeChatPayStrategy());
        context.pay(200.0);

        context.setStrategy(new CreditCardStrategy());
        context.pay(500.0);
    }
}
```

✅ 输出：

```
使用支付宝支付：100.0 元
使用微信支付：200.0 元
使用信用卡支付：500.0 元
```

💡 以后要加 ApplePay？
只需新建一个 `ApplePayStrategy` 类，无需改原逻辑。

## 💡 四、示例二：日志输出策略

日志可以输出到不同目标（控制台 / 文件 / 网络）。

```java
interface LogStrategy {
    void log(String msg);
}

class ConsoleLog implements LogStrategy {
    public void log(String msg) { System.out.println("[控制台] " + msg); }
}

class FileLog implements LogStrategy {
    public void log(String msg) { System.out.println("[文件写入] " + msg); }
}

class RemoteLog implements LogStrategy {
    public void log(String msg) { System.out.println("[远程服务上传] " + msg); }
}

class Logger {
    private LogStrategy strategy;
    public Logger(LogStrategy strategy) { this.strategy = strategy; }
    public void setStrategy(LogStrategy strategy) { this.strategy = strategy; }
    public void log(String msg) { strategy.log(msg); }
}

// 客户端
public class LogDemo {
    public static void main(String[] args) {
        Logger logger = new Logger(new ConsoleLog());
        logger.log("系统启动");

        logger.setStrategy(new FileLog());
        logger.log("保存到日志文件");

        logger.setStrategy(new RemoteLog());
        logger.log("上传远程监控中心");
    }
}
```

✅ 输出：

```
[控制台] 系统启动
[文件写入] 保存到日志文件
[远程服务上传] 上传远程监控中心
```

📘 客户端可灵活切换策略，而不用改核心逻辑。

## ⚙️ 五、示例三：排序策略（算法选择）

有多个排序算法（冒泡、快速、归并）。
我们用策略模式统一接口：

```java
interface SortStrategy {
    void sort(int[] arr);
}

class BubbleSort implements SortStrategy {
    public void sort(int[] arr) {
        System.out.println("使用冒泡排序");
        // 略...
    }
}

class QuickSort implements SortStrategy {
    public void sort(int[] arr) {
        System.out.println("使用快速排序");
        // 略...
    }
}

class MergeSort implements SortStrategy {
    public void sort(int[] arr) {
        System.out.println("使用归并排序");
        // 略...
    }
}

class SortContext {
    private SortStrategy strategy;
    public SortContext(SortStrategy strategy) { this.strategy = strategy; }
    public void setStrategy(SortStrategy strategy) { this.strategy = strategy; }
    public void execute(int[] arr) { strategy.sort(arr); }
}

// 客户端
public class SortDemo {
    public static void main(String[] args) {
        int[] arr = {5, 2, 8, 1};
        SortContext context = new SortContext(new BubbleSort());
        context.execute(arr);

        context.setStrategy(new QuickSort());
        context.execute(arr);
    }
}
```

✅ 输出：

```
使用冒泡排序
使用快速排序
```

💡 运行时选择不同算法，不用修改核心逻辑。

## 🧠 六、策略模式的本质

> **策略模式 = 把行为封装成独立策略对象，通过组合代替条件分支。**

它其实就是“用**多态 + 组合**取代`if-else`”。

---

## 🧩 七、与其他模式的区别

| 模式     | 目的                       | 区别                     |
| -------- | -------------------------- | ------------------------ |
| 模板方法 | 固定算法结构，让子类填步骤 | 继承 + 固定流程          |
| 策略模式 | 让算法完全可替换           | 组合 + 自由切换          |
| 状态模式 | 行为取决于状态变化         | 策略在外部选，状态自动切 |

> 策略模式更强调“行为选择”，
> 而模板方法强调“行为顺序”。

---

## ⚙️ 八、优缺点总结

| 优点             | 说明                                     |
| ---------------- | ---------------------------------------- |
| ✅ 开闭原则       | 新增策略不改旧代码                       |
| ✅ 消除 if-else   | 逻辑清晰，职责单一                       |
| ✅ 支持运行时切换 | 可配置、可注入                           |
| ⚠️ 缺点           | 策略类数量增多、上下文需理解策略协作方式 |

---

## 🧾 九、在框架中的实际应用

| 场景            | 策略接口                            | 具体策略                                   |
| --------------- | ----------------------------------- | ------------------------------------------ |
| Java Comparator | `Comparator<T>`                     | `NaturalOrder`, `ReverseOrder`             |
| Spring Cache    | `KeyGenerator`                      | `SimpleKeyGenerator`, `CustomKeyGenerator` |
| Spring Security | `AuthenticationProvider`            | 不同登录认证方式                           |
| JDK ThreadPool  | `RejectedExecutionHandler`          | `AbortPolicy`, `CallerRunsPolicy` 等       |
| 日志系统        | `Appender`                          | ConsoleAppender、FileAppender              |
| AI / 算法系统   | `Tokenizer`, `Embedding`, `Sampler` | 各种模型策略实现                           |

---

## ✅ 十、一句话总结

> 🧩 **策略模式（Strategy Pattern）**：
> 将算法或行为封装成可互换的策略对象，通过组合替代条件判断，使系统更灵活、更易扩展。
>
> 🧠 “用多态消灭 if-else，用策略实现可配置的行为。”
