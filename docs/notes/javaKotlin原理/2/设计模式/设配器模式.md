介绍下适配器模式并举几个例子

非常好 👍
 适配器模式（**Adapter Pattern**）是另一种常用的**结构型设计模式**，与装饰者模式有点像——都是**“包一层”**，但目的不同。
 装饰者是“增强功能”，而**适配器是“接口转换”**。

------

## 🧩 一、核心思想

适配器模式用于：

> **将一个类的接口转换成客户端所期望的另一个接口。**
>  让原本因为接口不兼容而不能一起工作的类可以协同工作。

换句话说，它是一个“中间层”，负责**接口匹配**。

------

## ⚙️ 二、主要角色

| 角色                    | 作用                                                 |
| ----------------------- | ---------------------------------------------------- |
| **Target（目标接口）**  | 客户端期望使用的接口                                 |
| **Adaptee（被适配者）** | 现有接口，功能正确但不兼容                           |
| **Adapter（适配器）**   | 实现 `Target` 接口并内部调用 `Adaptee`，完成接口转换 |

------

## 💡 三、示例 1：电源插头适配（最形象）

假设你有一个“220V电源”（被适配者），而手机充电器只支持“5V输入”（目标接口）。

```java
// 目标接口
interface DC5 {
    int output5V();
}

// 被适配者
class AC220 {
    public int output220V() {
        return 220;
    }
}

// 适配器
class PowerAdapter implements DC5 {
    private AC220 ac220;

    public PowerAdapter(AC220 ac220) {
        this.ac220 = ac220;
    }

    public int output5V() {
        int src = ac220.output220V();
        int dst = src / 44;
        System.out.println("适配器：将 " + src + "V 转为 " + dst + "V");
        return dst;
    }
}

// 客户端
public class Main {
    public static void main(String[] args) {
        DC5 adapter = new PowerAdapter(new AC220());
        int voltage = adapter.output5V();
        System.out.println("手机充电输入电压：" + voltage + "V");
    }
}
```

**输出：**

```
适配器：将 220V 转为 5V
手机充电输入电压：5V
```

👉 这就是典型的**类接口不兼容**问题，通过适配器解决。

## 💻 四、示例 2：Java 集合中的适配器

Java 自带大量适配器。
 比如 `Arrays.asList()` 方法：
 它将一个数组适配成 `List` 接口对象。

```java
String[] arr = {"A", "B", "C"};
List<String> list = Arrays.asList(arr); // 适配器
```

`Arrays.asList()` 内部其实返回了一个**适配器类**，它包装了数组，让数组表现得像一个 `List`。

## 📚 五、示例 3：日志系统统一接口

你可能在项目中使用不同的日志框架（Log4j、SLF4J、Logback），但你希望系统统一调用：

```java
interface Logger {
    void log(String message);
}

class Log4jLogger {
    public void write(String msg) {
        System.out.println("[Log4j] " + msg);
    }
}

class LogAdapter implements Logger {
    private Log4jLogger log4j;
    public LogAdapter(Log4jLogger log4j) { this.log4j = log4j; }
    public void log(String message) {
        log4j.write(message); // 接口适配
    }
}

public class Demo {
    public static void main(String[] args) {
        Logger logger = new LogAdapter(new Log4jLogger());
        logger.log("系统启动");
    }
}
```

**输出：**

```
[Log4j] 系统启动
```

👉 客户端使用统一的 `Logger` 接口，而底层可以无缝切换不同日志框架。

------

## 🔄 六、适配器的两种实现方式

| 类型                   | 说明                                   | 特点                         |
| ---------------------- | -------------------------------------- | ---------------------------- |
| **对象适配器（常用）** | 适配器中**持有被适配对象**引用（组合） | 灵活、安全                   |
| **类适配器（少用）**   | 适配器**继承被适配类**                 | 使用多继承，Java中限制多继承 |

------

## 🧠 七、优缺点

**优点：**

- 解耦客户端与不兼容类；
- 复用已有功能；
- 遵循开闭原则（对扩展开放，对修改封闭）。

**缺点：**

- 结构层次增多；
- 如果滥用会让系统复杂、难以理解。

------

## ✅ 八、适用场景

- 系统中存在旧接口或第三方库，需要与新接口兼容；
- 需要复用功能但接口定义不同；
- 不方便修改原类代码时（尤其是外部库）。

------
