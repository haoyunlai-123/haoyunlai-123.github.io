介绍下工厂模式并举几个例子

非常好 👍
 工厂模式（**Factory Pattern**）是最常见、最实用的**创建型设计模式**之一。
 它的核心思想是：

> **不直接在代码中 `new` 对象，而是把创建对象的逻辑交给“工厂类”来统一管理。**

这样可以让“创建”和“使用”分离，提高扩展性与解耦性。

------

## 🧩 一、为什么要用工厂模式？

如果直接写：

```java
Car car = new Bmw();  // new 在代码里到处都是
```

以后要改成：

```java
Car car = new Tesla();
```

那么所有地方都得改，非常不方便。
 工厂模式就是为了解决这种**“依赖具体类、修改成本高”**的问题。

------

## ⚙️ 二、工厂模式的三种主要形式

| 模式类型                         | 特点                                       | 场景                           |
| -------------------------------- | ------------------------------------------ | ------------------------------ |
| **简单工厂（Simple Factory）**   | 用一个工厂类根据参数创建不同对象           | 对象类型少，逻辑简单           |
| **工厂方法（Factory Method）**   | 抽象工厂定义创建接口，不同子类生产不同对象 | 扩展性强，新增类型无需改旧代码 |
| **抽象工厂（Abstract Factory）** | 一个工厂生产**一族**相关产品               | 复杂产品族场景（UI、数据库等） |

------

## 💡 三、简单工厂模式（最常用）

```java
// 产品接口
interface Car {
    void drive();
}

// 具体产品
class Tesla implements Car {
    public void drive() { System.out.println("开特斯拉！"); }
}

class Bmw implements Car {
    public void drive() { System.out.println("开宝马！"); }
}

// 工厂类
class CarFactory {
    public static Car createCar(String type) {
        if ("Tesla".equalsIgnoreCase(type)) {
            return new Tesla();
        } else if ("Bmw".equalsIgnoreCase(type)) {
            return new Bmw();
        }
        throw new IllegalArgumentException("未知类型");
    }
}

// 客户端
public class Main {
    public static void main(String[] args) {
        Car car = CarFactory.createCar("Tesla");
        car.drive();
    }
}
```

**输出：**

```
开特斯拉！
```

✅ 优点：客户端不需要知道 `new` 哪个类。
 ❌ 缺点：新增车型时要修改 `CarFactory` 的 `if` 逻辑，不符合开闭原则。

------

## 🏭 四、工厂方法模式（更符合开闭原则）

把“工厂”也抽象成接口，每种产品对应一个工厂实现：

```java
// 产品接口
interface Car {
    void drive();
}

// 具体产品
class Tesla implements Car {
    public void drive() { System.out.println("开特斯拉！"); }
}

class Bmw implements Car {
    public void drive() { System.out.println("开宝马！"); }
}

// 抽象工厂接口
interface CarFactory {
    Car createCar();
}

// 具体工厂
class TeslaFactory implements CarFactory {
    public Car createCar() { return new Tesla(); }
}

class BmwFactory implements CarFactory {
    public Car createCar() { return new Bmw(); }
}

// 客户端
public class Demo {
    public static void main(String[] args) {
        CarFactory factory = new TeslaFactory();  // 多态切换
        Car car = factory.createCar();
        car.drive();
    }
}
```

✅ 优点：

- 新增车型只需添加一个新的工厂类；
- 原代码不动，符合 **开闭原则**。

------

## 🧰 五、抽象工厂模式（生产“产品族”）

当一个工厂不止生产一个对象时，比如同时生产“汽车 + 轮胎 + 引擎”。

```java
// 产品族接口
interface Engine { void start(); }
interface Tire { void roll(); }

// 具体产品
class TeslaEngine implements Engine { public void start() { System.out.println("特斯拉电机启动"); } }
class TeslaTire implements Tire { public void roll() { System.out.println("特斯拉轮胎滚动"); } }

class BmwEngine implements Engine { public void start() { System.out.println("宝马引擎启动"); } }
class BmwTire implements Tire { public void roll() { System.out.println("宝马轮胎滚动"); } }

// 抽象工厂
interface CarFactory {
    Engine createEngine();
    Tire createTire();
}

// 具体工厂
class TeslaFactory implements CarFactory {
    public Engine createEngine() { return new TeslaEngine(); }
    public Tire createTire() { return new TeslaTire(); }
}

class BmwFactory implements CarFactory {
    public Engine createEngine() { return new BmwEngine(); }
    public Tire createTire() { return new BmwTire(); }
}

// 客户端
public class Demo {
    public static void main(String[] args) {
        CarFactory factory = new TeslaFactory();
        factory.createEngine().start();
        factory.createTire().roll();
    }
}
```

➡️ 一个工厂可以生产一整套相互匹配的产品。

------

## 🧠 六、三者对比总结

| 模式         | 创建逻辑位置             | 新增产品代价   | 特点             |
| ------------ | ------------------------ | -------------- | ---------------- |
| **简单工厂** | 集中在一个工厂类中       | 改动原工厂     | 代码简单但不灵活 |
| **工厂方法** | 每种产品独立工厂         | 新增一个工厂   | 符合开闭原则     |
| **抽象工厂** | 一个工厂生产多个相关产品 | 新增产品族复杂 | 适用于产品族系统 |

------

## 💻 七、现实应用举例

| 场景                   | 示例                                                         |
| ---------------------- | ------------------------------------------------------------ |
| **JDBC 驱动加载**      | `DriverManager.getConnection()` 内部根据不同数据库驱动动态返回连接对象（工厂方法） |
| **Spring BeanFactory** | IOC 容器根据配置创建 Bean 实例（抽象工厂）                   |
| **日志框架**           | `LoggerFactory.getLogger()` 根据配置返回不同日志实现         |
| **GUI 框架**           | WindowsFactory / MacFactory 生成不同平台的 UI 控件（抽象工厂） |

------

## ✅ 八、优缺点

**优点：**

- 解耦创建与使用；
- 方便扩展；
- 让对象创建可配置化（尤其在框架中）。

**缺点：**

- 类层次增加；
- 简单业务可能显得“过度设计”。
