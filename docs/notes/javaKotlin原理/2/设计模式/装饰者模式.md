介绍下装饰者模式并举几个例子

装饰者模式（**Decorator Pattern**）是结构型设计模式的一种，它用于**在不修改原类的前提下，为对象动态地添加功能**。

------

## 🧩 一、核心思想

装饰者模式通过**将对象放入一个包装对象（装饰者）中**，来增强原对象的行为。
 装饰者与被装饰对象实现相同的接口，因此**外部使用时无需关心对象是否被装饰**，从而实现“即插即用”的功能扩展。

------

## ⚙️ 二、结构组成

| 角色                                | 作用                                    |
| ----------------------------------- | --------------------------------------- |
| **Component（抽象构件）**           | 定义对象的统一接口                      |
| **ConcreteComponent（具体构件）**   | 被装饰的原始对象                        |
| **Decorator（抽象装饰者）**         | 持有一个 Component 引用，并实现相同接口 |
| **ConcreteDecorator（具体装饰者）** | 扩展原始对象的功能                      |

------

## 💡 三、示例 1：咖啡加配料（经典例子）

```java
// 抽象组件
interface Coffee {
    String getDescription();
    double cost();
}

// 具体组件
class SimpleCoffee implements Coffee {
    public String getDescription() { return "Simple coffee"; }
    public double cost() { return 5.0; }
}

// 抽象装饰者
abstract class CoffeeDecorator implements Coffee {
    protected Coffee decoratedCoffee;
    public CoffeeDecorator(Coffee coffee) { this.decoratedCoffee = coffee; }
    public String getDescription() { return decoratedCoffee.getDescription(); }
    public double cost() { return decoratedCoffee.cost(); }
}

// 具体装饰者
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) { super(coffee); }
    public String getDescription() { return decoratedCoffee.getDescription() + ", milk"; }
    public double cost() { return decoratedCoffee.cost() + 2.0; }
}

class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) { super(coffee); }
    public String getDescription() { return decoratedCoffee.getDescription() + ", sugar"; }
    public double cost() { return decoratedCoffee.cost() + 1.0; }
}

public class Main {
    public static void main(String[] args) {
        Coffee coffee = new SimpleCoffee();
        coffee = new MilkDecorator(coffee);
        coffee = new SugarDecorator(coffee);
        System.out.println(coffee.getDescription() + " => $" + coffee.cost());
    }
}
```

**输出：**

```
Simple coffee, milk, sugar => $8.0
```

👉 在不修改 `SimpleCoffee` 的情况下，动态叠加“牛奶”“糖”的功能。

## 💻 四、示例 2：输入流增强（Java I/O 的真实应用）

Java 的 I/O 库大量使用了装饰者模式。

```java
InputStream input = new FileInputStream("data.txt");
InputStream buffered = new BufferedInputStream(input);
InputStream data = new DataInputStream(buffered);
int value = data.read();
```

- `FileInputStream`：基础组件（读取字节）
- `BufferedInputStream`：装饰者（增加缓冲功能）
- `DataInputStream`：装饰者（支持读原始类型数据）

👉 各层装饰器都实现 `InputStream` 接口，形成功能叠加链。

## 🔒 五、示例 3：Web 请求日志增强（装饰者在框架中的应用）

在 Spring 中也常见到类似思想：

```java
interface Handler {
    void handle();
}

class SimpleHandler implements Handler {
    public void handle() { System.out.println("处理请求"); }
}

class LogHandler implements Handler {
    private Handler target;
    public LogHandler(Handler target) { this.target = target; }
    public void handle() {
        System.out.println("记录日志...");
        target.handle();
        System.out.println("请求完成");
    }
}

public class Demo {
    public static void main(String[] args) {
        Handler handler = new LogHandler(new SimpleHandler());
        handler.handle();
    }
}
```

------

## 🧠 六、优缺点总结

**优点：**

- 动态扩展对象功能，不修改原类；
- 可组合多个装饰者，灵活性高；
- 符合开闭原则（OCP）。

**缺点：**

- 对象层次过多，调试困难；
- 小对象数量增多，可能增加系统复杂度。

------

## ✅ 七、适用场景

- 想在**运行时动态添加功能**；
- 不希望使用继承扩展功能；
- 需要多种功能组合（如日志 + 缓存 + 权限控制）。
