非常好 👍 这道题你问得非常经典，
它正是 Java **类初始化顺序 + 编译器“前向引用检查”机制（Illegal forward reference）** 的典型案例。
我们来逐行解析，让你彻底理解为什么会报错、为什么 `num` 却可以访问，而 `number` 不行。

---

## 🧩 一、首先看你这段代码的顺序

```java
public class Test {
    private static int num = 1;

    static {
        num = 2;
        number = 10;
        System.out.println(num);
        System.out.println(number); // ❌ 报非法前向引用
    }

    private static int number = 20;
}
```

---

## 🧠 二、编译器为什么报错（“非法前向引用”）

错误行是：

```java
System.out.println(number);
```

编译器错误信息类似：

```
illegal forward reference
```

意思是：

> 在类的**静态初始化块**中，访问了一个**尚未声明并初始化的静态变量**。

---

## 🧩 三、先搞清楚“类初始化”的两个阶段

在 Java 中，类中定义的静态成员变量和静态代码块**在编译期会被合并成一个按源代码顺序执行的初始化方法**（称为 *<clinit>* 方法）。

我们来看看编译器眼中的顺序（伪代码）：

```java
class Test {
    static int num;
    static int number;

    static {
        num = 1;
        num = 2;
        number = 10;
        System.out.println(num);
        System.out.println(number);
        number = 20;
    }
}
```

而编译器执行检查的时候会分两步考虑：

1. 在类的**文本顺序（源代码顺序）**中逐条扫描；
2. 如果在**变量声明之前**访问了该变量（且不是简单赋值），就会报 “非法前向引用”。

---

## ⚙️ 四、为什么 `num` 可以访问而 `number` 不行？

### ✅ `num` 的情况

```java
private static int num = 1;

static {
    num = 2;
    System.out.println(num);
}
```

* `num` 的声明出现在静态块**之前**；
* 因此静态块中访问它是合法的；
* 它在执行 `<clinit>` 方法时已经在常量池中分配了空间。

✅ 所以 `num` 可以被读写。

---

### ❌ `number` 的情况

```java
static {
    number = 10;                // ✅ 合法（赋值操作允许）
    System.out.println(number); // ❌ 非法（读取操作不允许）
}

private static int number = 20;
```

* `number` 声明在静态块**之后**；
* 此时编译器在扫描到 `System.out.println(number)` 时，
  它认为 `number` 还没有“被定义”；
* 因为类还没走到定义那一行，所以禁止提前访问；
* 但**赋值语句**（如 `number = 10;`）是特殊允许的；
  它被认为是类变量的“定义阶段的一部分”。

---

## 🧩 五、所以“非法前向引用”的判定规则是：

| 操作类型                                         | 是否允许访问未定义在前的静态变量 |
| -------------------------------------------- | ---------------- |
| ✅ **赋值**（`number = 10;`）                     | 允许（编译器会延迟绑定）     |
| ❌ **取值 / 读取**（`System.out.println(number);`） | 不允许（非法前向引用）      |

---

## 🧠 六、背后原理：编译期规则而非运行期行为

这不是 JVM 报错，而是 **Java 编译器（javac）** 在语义分析阶段检查出的错误。
JVM 实际上不会管“顺序”，但 Java 语言规范（JLS 8.3.3）明确规定：

> “在类或接口的静态上下文中，除非变量在文本上出现在当前点之前，否则不允许读取该变量的值。”

---

## 📘 七、正确写法（避免前向引用）

### ✅ 写法 1：调整声明顺序

```java
public class Test {
    private static int num = 1;
    private static int number = 20;

    static {
        num = 2;
        number = 10;
        System.out.println(num);
        System.out.println(number);
    }
}
```

### ✅ 写法 2：在静态块中只赋值，不读取

```java
public class Test {
    private static int num = 1;

    static {
        num = 2;
        number = 10;  // ✅ 赋值可以
    }

    private static int number = 20;
}
```

## ✅ 九、总结一句话

| 变量位置           | 操作类型   | 是否允许 | 原因                                |
| -------------- | ------ | ---- | --------------------------------- |
| 变量定义在静态块**之前** | 读取或写入  | ✅    | 变量已声明                             |
| 变量定义在静态块**之后** | 仅赋值    | ✅    | 编译器允许提前初始化                        |
| 变量定义在静态块**之后** | 读取（取值） | ❌    | 非法前向引用（Illegal forward reference） |

---

> 在类的静态初始化过程中，
> 编译器按源代码顺序处理静态变量和静态块。
> 在声明之前只能“赋值”，不能“读取”，
> 否则会被认为是“非法前向引用”。

## ⚠️ 一、编译错误的真正原因

错误的关键是：

> 你在 **静态初始化块**（或静态字段初始化）中引用了一个尚未定义完成的静态字段。

Java 的编译器在解析类时，会自上而下地“确定声明顺序”。
此时：

1. 编译器刚看到 `public static final Map<String, Document> registry;`
   → 它知道有这个字段，但它 **还没被赋值，也没有类型初始化**。

2. 紧接着编译器进入静态内部类 `Inner`。
   在这个类中，你引用 `PrototypeRegistry.registry`。
   但这时候对于编译器来说，`PrototypeRegistry.registry` **还未完全解析完成**，因为外部类本身的静态上下文还在构建。

3. 所以编译器认为这是一个 “非法的前向引用”（Illegal forward reference）。

---

## 🧠 二、类加载 & 初始化的时序（重点）

类加载顺序可以简化为：

1. **加载（Load）**：JVM 读取字节码
2. **连接（Link）**

   * 验证
   * 准备（静态字段分配默认值）
   * 解析
3. **初始化（Initialize）**：执行静态初始化块、字段赋值

你的 `PrototypeRegistry` 类在 **准备阶段** 仅仅分配了一个 `registry` 字段的空间；
但 **初始化阶段** 还没开始执行赋值逻辑。

而内部类 `Inner` 在加载时会主动引用外部类（形成依赖关系），
JVM 会先确保外部类已加载，但不保证初始化顺序完全完成。

👉 所以此时在静态上下文中引用 `PrototypeRegistry.registry`，
编译器无法保证它已可用。

---

## ✅ 三、正确写法 1：立即初始化静态字段

最简单的解决方法：直接初始化 `registry`。

```java
public class PrototypeRegistry {

    public static final Map<String, Document> registry = new HashMap<>();

    private static class Inner {
        static final PrototypeRegistry prototypeRegistry = new PrototypeRegistry();
    }

    public static PrototypeRegistry getInstance() {
        return Inner.prototypeRegistry;
    }

    public Document getDocument(String id) {
        return registry.get(id);
    }
}
```

💡 这里：

* `registry` 一开始就被初始化；
* 不存在“前向引用”问题；
* `Inner` 也能安全使用。

---

## ✅ 四、正确写法 2：在构造函数中初始化

如果你需要在创建 `PrototypeRegistry` 时初始化 `registry`，可以这样：

```java
public class PrototypeRegistry {

    public static final Map<String, Document> registry;

    static {
        registry = new HashMap<>();
    }

    private static class Inner {
        static final PrototypeRegistry prototypeRegistry = new PrototypeRegistry();
    }

    public static PrototypeRegistry getInstance() {
        return Inner.prototypeRegistry;
    }

    public Document getDocument(String id) {
        return registry.get(id);
    }
}
```

静态代码块保证 `registry` 初始化时机与类初始化同步。

---

## ✅ 五、正确写法 3：如果你的 registry 需要依赖实例初始化

那你就不能在静态上下文使用它，而应该让内部类引用外部实例的字段：

```java
public class PrototypeRegistry {

    private final Map<String, Document> registry = new HashMap<>();

    private static class Inner {
        static final PrototypeRegistry INSTANCE = new PrototypeRegistry();
    }

    public static PrototypeRegistry getInstance() {
        return Inner.INSTANCE;
    }

    public Document getDocument(String id) {
        return registry.get(id);
    }
}
```

📘 优点：

* 不需要静态字段；
* 仍然是线程安全的单例（静态内部类懒加载）；
* 符合原型注册中心（Prototype Registry）的职责。

| 写法     | registry 声明         | 初始化时机    | 是否可在 Inner 中使用 |
| ------ | ------------------- | -------- | -------------- |
| ❌ 原始写法 | 未初始化静态字段            | 尚未完成类初始化 | 否              |
| ✅ 写法 1 | `= new HashMap<>()` | 定义时初始化   | 可以             |

## ✅ 七、一句话总结

> “`super.clone()` 是为复制继承链；
> `super.<field>` 在静态上下文中若引用未初始化字段，会触发非法前向引用。”
>
> 在你的例子中，**`registry` 还没初始化完成就被静态内部类引用**，
> 所以编译器拒绝。只要提前初始化（或改为实例字段）即可。
