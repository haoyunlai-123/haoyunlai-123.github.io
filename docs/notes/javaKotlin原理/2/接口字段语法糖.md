public interface Test {
    
    String a = "123";

}接口中为什么能有成员变量？

接口里的“成员变量”**并不是实例字段**，而是：

> **public static final 的常量** —— 编译器会自动加上这 3 个修饰符，因此写不写都一样。

------------------------------------------------
1. 编译器自动补全（反编译看真相）
```java
public interface Test {
    String a = "123";
}
```
反编译后：
```java
public interface Test {
    public static final String a = "123";
}
```
- **public**：接口内默认全局可见  
- **static**：属于接口本身，不属于任何实现类实例  
- **final**：必须初始化且不可重新赋值

------------------------------------------------
2. 为什么不能放“普通成员变量”？
接口**不能被实例化**，也没有构造器，因此**没有对象状态**；  
若允许实例字段，就必须有构造器/初始化块——与接口设计初衷冲突。

------------------------------------------------
3. 使用场景
- **定义常量**（推荐用枚举或类常量，但接口常量语法合法）：
  ```java
  public interface Constants {
      int MAX_SIZE = 100;
  }
  ```
- **策略模式**里把实现类注册名写成常量：
  ```java
  public interface PaymentStrategy {
      String ALI = "aliPay";
      String WECHAT = "wechatPay";
  }
  ```

------------------------------------------------
4. 一句话记忆
> **接口里的“变量”= public static final 常量；**  
> **编译器自动加 3 个关键字，因此语法上看起来像“成员变量”，实质是全局常量。**
