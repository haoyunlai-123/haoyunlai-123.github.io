写个方法将当前user.dir下的a.zip解压缩到user.dir下

可以，下面给出一个简洁、标准库即可运行的解压方法，
 会将当前工作目录（`user.dir`）下的 `a.zip` 解压到同一目录下（即 `user.dir`）。

------

### ✅ 完整示例代码

```java
import java.io.*;
import java.nio.file.*;
import java.util.zip.*;

public class UnzipExample {

    public static void main(String[] args) throws IOException {
        unzipToUserDir();
    }

    /**
     * 将当前 user.dir 下的 a.zip 解压到 user.dir 下
     */
    public static void unzipToUserDir() throws IOException {
        // 获取当前工作目录
        String userDir = System.getProperty("user.dir");
        Path zipPath = Paths.get(userDir, "a.zip");
        Path destDir = Paths.get(userDir);

        // 校验 zip 文件是否存在
        if (!Files.exists(zipPath)) {
            throw new FileNotFoundException("未找到文件: " + zipPath);
        }

        // 打开 zip 文件
        try (ZipInputStream zis = new ZipInputStream(new FileInputStream(zipPath.toFile()))) {
            ZipEntry entry;
            while ((entry = zis.getNextEntry()) != null) {
                Path newPath = destDir.resolve(entry.getName()).normalize(); // 目标路径

                // 防止 Zip Slip 攻击
                if (!newPath.startsWith(destDir)) {
                    throw new IOException("非法条目: " + entry.getName());
                }

                if (entry.isDirectory()) {
                    Files.createDirectories(newPath);
                } else {
                    // 创建父目录
                    Files.createDirectories(newPath.getParent());
                    // 写入文件
                    try (OutputStream os = Files.newOutputStream(newPath)) {
                        zis.transferTo(os);
                    }
                }
                zis.closeEntry();
            }
        }

        System.out.println("✅ 解压完成: " + destDir);
    }
}
```

------

### 💡 特点说明

| 特点                                  | 说明                                          |
| ------------------------------------- | --------------------------------------------- |
| ✅ 使用 `java.util.zip.ZipInputStream` | 无需任何第三方库                              |
| ✅ 自动创建目录                        | 确保子目录结构完整                            |
| ✅ 防止 Zip Slip                       | 通过 `normalize()` + 路径检查防御目录穿越攻击 |
| ✅ 支持中文文件名                      | JDK 11+ 默认支持 UTF-8 zip entry              |

Path.normalize()这个是干啥的？

`Path.normalize()` 就是“**路径瘦身器**”——把路径里的 `.`（当前目录）和 `..`（上级目录）**全部消化掉**，返回**等价但最简**的 Path 对象，**不做任何磁盘访问**。

示例（Linux/Windows 通用）：

```java
Path p = Paths.get("/home/./foo/../bar/./readme.txt");
Path clean = p.normalize();   // → /home/bar/readme.txt
```

特点  
- **纯字符串级**运算，不检查文件是否存在；  
- 多余分隔符、`./`、 trailing `/` 也会被去掉；  
- 若 `..` 溢出根（如 `a/b/../../..`），结果保留最左侧 `..`（不会抛异常）。

一句话：  
**normalize = 把“点点杠”全部吃掉，让路径最短、最顺眼。**
