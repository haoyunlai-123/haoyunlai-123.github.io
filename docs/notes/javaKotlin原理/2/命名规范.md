java中包名的命名规范是什么？

Java 包名全部 **小写**，用 **点号** 分隔，**逆序的 DNS 域名** 开头，后面跟项目/模块/分层名。  
一句话：**“反域名，全小写，不下滑，不特殊”**。

具体细则：

1. 基本格式  
   `com.example.product.module.layer`

2. 字符规则  
   - 只允许 `a-z`、数字、点号。  
   - **禁止下划线 `_`、连字符 `-`、美元符 `$`**。  
   - 数字不能放在段首（`com.1st` 非法）。

3. 域名段顺序  
   把真实域名倒过来：  
   `cn.mycorp.project.app`  
   没有域名时，用 `io.github.用户名`、`net.作者名` 或 `org.组织名` 代替。

4. 分层/模块命名  
   常见后缀（按层）：  
   - `.api`  
   - `.service`  
   - `.dao` / `.repository`  
   - `.model` / `.domain`  
   - `.web` / `.controller`  
   - `.util`  
   - `.common`  

   按模块：  
   - `.order`  
   - `.payment`  
   - `.user`  

   组合示例：  
   `com.shop.order.api`  
   `com.shop.order.service`  
   `com.shop.order.dao`

5. 单级包名（默认包）  
   禁止把类直接放在默认包（无包名）里，既无法被其它包导入，也无法在模块化项目中编译。

6. 示例对照  
   ✅ 合法  
   ```
   com.alipay.tinyapp.common.util
   io.github.johndoe.jsonparser.core
   cn.gov.customs.risk.dao
   ```

   ❌ 非法  
   ```
   com.my-corp.util      （含连字符）
   com.myCorp.Util       （含大写）
   com.my_corp.util      （含下划线）
   com.3rdparty.lib      （数字开头）
   ```

记住：  
**反域名、全小写、见名知义、逐级细分**，就是 Java 包名的唯一规范。

包名有多个单词呢？

包名里**不会出现“多个单词”的驼峰或下划线写法**，一律**全部小写、直接连写**，需要分段就再点一级。  
换句话说：**用“额外的包级”代替单词分隔符**。

举例：

| 业务含义     | 推荐包名                      | 错误示例                       |
| ------------ | ----------------------------- | ------------------------------ |
| 用户服务     | `com.shop.user.service`       | `com.shop.userService` ❌       |
| 订单支付核心 | `com.shop.order.payment.core` | `com.shop.orderPayment.core` ❌ |
| 通用工具     | `com.shop.common.util`        | `com.shop.commonUtil` ❌        |
| 微信 SDK     | `io.github.foo.wxsdk`         | `io.github.foo.wxSdk` ❌        |

要点回顾  
1. 每段只能一个小写单词。  
2. 想表达“多个单词”就再拆一层包，而不是用驼峰或下划线。  
3. 保持“反域名 + 模块/分层 + 子模块”的递进结构即可。
