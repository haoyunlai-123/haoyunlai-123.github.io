# 🧩 一、阻塞队列的定义

在 JUC（`java.util.concurrent`）包中，`BlockingQueue` 是一个接口：

```java
public interface BlockingQueue<E> extends Queue<E> {
    void put(E e) throws InterruptedException;
    E take() throws InterruptedException;
}
```

> 它是一种**支持线程阻塞的队列**：
>
> * 当队列满时，生产者线程会阻塞（等待有空位）；
> * 当队列空时，消费者线程会阻塞（等待有数据）。

---

# 🧠 二、主要实现类

| 实现类                  | 特点                     | 底层结构                              |
| ----------------------- | ------------------------ | ------------------------------------- |
| `ArrayBlockingQueue`    | 有界、数组结构、可公平锁 | `ReentrantLock + Condition`           |
| `LinkedBlockingQueue`   | 可选容量、链表结构       | `ReentrantLock + Condition`（两把锁） |
| `PriorityBlockingQueue` | 基于优先级堆             | 无界，不阻塞 put()                    |
| `DelayQueue`            | 支持延时取出             | 基于优先队列 + 时间                   |
| `SynchronousQueue`      | 无容量，直接交接         | `TransferQueue` 实现                  |
| `LinkedTransferQueue`   | 高性能队列               | CAS + LockSupport                     |

# ⚙️ 三、原理核心 —— “两种阻塞条件”

以最常用的 `ArrayBlockingQueue` 为例：

* 内部维护一个 **数组 `items[]`**；
* 使用两个 `Condition` 对象：

  * `notEmpty`：队列为空时消费者等待；
  * `notFull`：队列满时生产者等待；
* 锁使用 `ReentrantLock`。

源码结构（精简）：

```java
final ReentrantLock lock = new ReentrantLock();
private final Condition notEmpty = lock.newCondition();
private final Condition notFull = lock.newCondition();

public void put(E e) throws InterruptedException {
    final ReentrantLock lock = this.lock;
    lock.lockInterruptibly();
    try {
        while (count == items.length)
            notFull.await();   // 队列满 → 生产者阻塞
        enqueue(e);
        notEmpty.signal();     // 唤醒等待的消费者
    } finally {
        lock.unlock();
    }
}

public E take() throws InterruptedException {
    final ReentrantLock lock = this.lock;
    lock.lockInterruptibly();
    try {
        while (count == 0)
            notEmpty.await();  // 队列空 → 消费者阻塞
        E x = dequeue();
        notFull.signal();      // 唤醒等待的生产者
        return x;
    } finally {
        lock.unlock();
    }
}
```

# 🔄 四、阻塞与唤醒的底层原理

关键在 `Condition` 的 `await()` 和 `signal()`：

### 1️⃣ 当线程调用 `await()` 时：

* 当前线程持有锁；
* 执行流程：

  1. 当前线程从 **AQS 队列（锁等待队列）** 转移到 **Condition 队列**；
  2. 释放锁；
  3. 阻塞挂起（通过 `LockSupport.park()`）；
  4. 等待 `signal()` 唤醒。

### 2️⃣ 当其他线程调用 `signal()` 时：

* 唤醒一个等待在 `Condition` 上的线程；
* 被唤醒的线程进入 **AQS 锁等待队列**；
* 当它重新获得锁后，`await()` 方法返回。

---

🔧 **底层机制图解：**

```
[Producer Thread]                        [Consumer Thread]
      ↓                                         ↓
 put() → while(full) → notFull.await()     take() → while(empty) → notEmpty.await()
      ↓                                         ↓
     signal() <-------------------------------< signal()
```

**await → park，signal → unpark**
所有这些操作由 AQS（AbstractQueuedSynchronizer）管理。

# 🧩 五、重要内部结构

`ConditionObject` 内部包含两个关键队列：

* **Condition Queue**：存放正在等待的线程；
* **AQS Sync Queue**：用于重新竞争锁。

唤醒过程如下：

1. `signal()` 将 `Condition Queue` 头节点转移到 `Sync Queue`；
2. 调用 `LockSupport.unpark(thread)`；
3. 被唤醒线程重新尝试获取锁。

这也是为什么：

> * `signal()` 不会立刻让另一个线程执行；
> * 它只是“激活”另一个线程，等锁释放后再运行。

# 🧵 七、示例：生产者-消费者模型

```java
public class ProducerConsumerDemo {
    private static final BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(5);

    static class Producer implements Runnable {
        public void run() {
            try {
                for (int i = 0; i < 10; i++) {
                    queue.put(i);
                    System.out.println("生产：" + i);
                    Thread.sleep(500);
                }
            } catch (InterruptedException e) { e.printStackTrace(); }
        }
    }

    static class Consumer implements Runnable {
        public void run() {
            try {
                while (true) {
                    int val = queue.take();
                    System.out.println("消费：" + val);
                    Thread.sleep(800);
                }
            } catch (InterruptedException e) { e.printStackTrace(); }
        }
    }

    public static void main(String[] args) {
        new Thread(new Producer()).start();
        new Thread(new Consumer()).start();
    }
}
```

# ⚙️ 八、与线程池的关系

`ThreadPoolExecutor` 内部就是基于 `BlockingQueue` 实现任务调度：

```java
public class ThreadPoolExecutor {
    private final BlockingQueue<Runnable> workQueue;
}
```

流程：

* `execute(task)` → 尝试放入队列；
* 若队列满 → 创建新线程；
* 若线程数达到上限 → 执行拒绝策略。

常见默认实现是：

```java
new LinkedBlockingQueue<>();
```

# 🚀 十、简要流程（take 操作）

```
[1] acquire lock
[2] if empty → await() → release lock + park()
[3] 被 signal() 唤醒 → 重新加入 AQS 等待队列
[4] 重新获得锁 → 出队元素 → unlock()
```

# ✅ 一、阻塞队列是什么（一句话理解）

> 一个**支持自动等待**的线程安全队列。
> 当队列**满**时，`put()` 会自动阻塞生产者；
> 当队列**空**时，`take()` 会自动阻塞消费者。

适用于：**生产者-消费者模型**、**线程池**、**消息传递**等。

| 实现类                  | 有界/无界              | 特点                 | 适用场景             |
| ----------------------- | ---------------------- | -------------------- | -------------------- |
| **ArrayBlockingQueue**  | ✅ 有界                 | 数组结构，性能稳定   | 控制容量、固定任务量 |
| **LinkedBlockingQueue** | ✅ 默认无界，可指定容量 | 链表结构，吞吐量高   | 线程池、日志队列     |
| **SynchronousQueue**    | ❌ 无容量               | 必须“交接”成功才继续 | 任务直接交给工作线程 |

# 🧩 三、基本使用示例

## ✅ 1. 生产者 - 消费者模型（经典）

```java
import java.util.concurrent.*;

public class BlockingQueueDemo {
    // 有界阻塞队列，最多放5个
    private static final BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(5);

    public static void main(String[] args) {
        new Thread(new Producer()).start();
        new Thread(new Consumer()).start();
    }

    static class Producer implements Runnable {
        public void run() {
            try {
                for (int i = 1; i <= 10; i++) {
                    queue.put(i);  // 队列满则阻塞
                    System.out.println("生产：" + i);
                    Thread.sleep(300);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    static class Consumer implements Runnable {
        public void run() {
            try {
                while (true) {
                    Integer item = queue.take();  // 队列空则阻塞
                    System.out.println("消费：" + item);
                    Thread.sleep(800);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}
```

运行效果：

```
生产：1
消费：1
生产：2
生产：3
消费：2
消费：3
...
```

👉 `put()` 会在满时阻塞，`take()` 会在空时阻塞，非常安全。

## ✅ 2. 非阻塞方法（可用在高并发系统）

有时你不希望线程真的阻塞，可以用 `offer()` + 超时：

```java
boolean success = queue.offer(100, 2, TimeUnit.SECONDS);
if (!success) {
    System.out.println("2秒内队列没空位，放弃提交");
}
```

* 如果队列满 → 等待 2 秒；
* 若还没空间 → 返回 `false`；
* 不会抛异常，不会永久阻塞。

同理消费端可以用：

```java
Integer item = queue.poll(1, TimeUnit.SECONDS);
if (item == null) {
    System.out.println("1秒内没有数据，继续下一轮");
}
```

## ✅ 3. 在线程池中使用（非常常见）

线程池内部默认就用了阻塞队列来存任务：

```java
ExecutorService pool = new ThreadPoolExecutor(
    2,                  // 核心线程数
    5,                  // 最大线程数
    10L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(10)  // 阻塞队列
);
```

使用方式：

```java
for (int i = 0; i < 20; i++) {
    final int taskId = i;
    pool.execute(() -> {
        System.out.println("执行任务：" + taskId);
    });
}
```

👉 当线程满时，新任务会进入队列等待；
👉 队列也满时，则会触发拒绝策略。

## ✅ 4. SynchronousQueue —— “手递手”队列

`SynchronousQueue` **不存储任何元素**，必须一手交一手：

```java
BlockingQueue<String> queue = new SynchronousQueue<>();

new Thread(() -> {
    try {
        System.out.println("生产1");
        queue.put("A");
        System.out.println("生产完成");
    } catch (InterruptedException e) { }
}).start();

new Thread(() -> {
    try {
        Thread.sleep(1000);
        System.out.println("消费：" + queue.take());
    } catch (InterruptedException e) { }
}).start();
```

输出：

```
生产1
消费：A
生产完成
```

👉 生产线程会一直等，直到有消费者取走。
常用于**直接交接任务**（例如 `newCachedThreadPool()` 使用的就是它）。

---

## ✅ 5. PriorityBlockingQueue —— 按优先级取元素

```java
BlockingQueue<Integer> queue = new PriorityBlockingQueue<>();
queue.put(5);
queue.put(1);
queue.put(3);

System.out.println(queue.take()); // 输出 1
System.out.println(queue.take()); // 输出 3
System.out.println(queue.take()); // 输出 5
```

👉 内部基于堆结构，自动按优先级出队。

# 🔁 四、总结：四类操作方式（记住就行）

| 操作类型 | 抛异常 | 返回特殊值 | 阻塞等待 | 超时等待 |
|-----------|----------|--------------|--------------|
| 插入 | `add(e)` | `offer(e)` | `put(e)` | `offer(e, time, unit)` |
| 移除 | `remove()` | `poll()` | `take()` | `poll(time, unit)` |
| 检查 | `element()` | `peek()` | - | - |

> 一般开发推荐：
>
> * **生产者用 `put()` 或 `offer(timeout)`**
> * **消费者用 `take()` 或 `poll(timeout)`**
