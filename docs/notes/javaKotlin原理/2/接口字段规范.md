为什么接口中不能声明private和protected类型?

接口的设计定位是“**对外契约**”，所有成员都必须是**对实现类可见**的；  
`private` 和 `protected` 的可见性与此冲突，所以语法层面直接禁止。

1. 语义冲突  
   - `private`：仅本类可见；接口没有“实现体”，实现类无法访问，也就无法重写或调用。  
   - `protected`：包内 + 子类可见；接口没有“子类”概念，实现类与接口往往不在同一包，也无法保证一定能访问。

2. 历史与兼容性  
   Java 1.0 起接口就是“**100% public abstract**”的纯抽象规格；  
   如果后来允许 `private/protected`，所有老实现类都会突然“看不到”这些成员，破坏二进制兼容性。

3. 技术实现  
   接口的字节码要求所有方法标志位为 `ACC_PUBLIC | ACC_ABSTRACT`（或 `ACC_STATIC`、`ACC_PRIVATE` 仅用于 static 私有辅助方法，见下文）。  
   编译器在解析阶段就会拒绝 `protected` 和“实例 private”方法。

例外（仅 static）  
Java 9 开始接口可以写  
```java
private static void helper() { … }
```
因为它是**静态辅助方法**，不涉及实现类，也不会被继承，所以不违反“对外契约”原则。  
但 **private/protected 实例方法** 至今仍被禁止。
