好的，我们来梳理一下 **Java 元注解 @Target 和 @Retention 的属性**。

------

## 1. @Target

- **作用**：限定一个自定义注解可以用在什么位置。

- 属性

  ：

  - ```
    ElementType[] value();
    ```

    - 类型是一个数组，元素来自枚举 `java.lang.annotation.ElementType`。
    - 常见取值：
      - `TYPE`：类、接口（包括注解类型）、枚举
      - `FIELD`：字段（包括枚举常量）
      - `METHOD`：方法
      - `PARAMETER`：方法参数
      - `CONSTRUCTOR`：构造函数
      - `LOCAL_VARIABLE`：局部变量
      - `ANNOTATION_TYPE`：注解类型
      - `PACKAGE`：包
      - `TYPE_PARAMETER`（Java 8+）：类型参数
      - `TYPE_USE`（Java 8+）：类型使用位置（泛型、强转、implements 等）
      - `MODULE`（Java 9+）：模块声明

👉 示例：

```java
@Target({ElementType.METHOD, ElementType.TYPE})
public @interface MyAnno {}
```

表示 `@MyAnno` 只能用在类和方法上。

------

## 2. @Retention

- **作用**：指定注解的生命周期（保留到哪个阶段）。

- 属性

  ：

  - ```
    RetentionPolicy value();
    ```

    - 类型是枚举 `java.lang.annotation.RetentionPolicy`。
    - 取值：
      - `SOURCE`：只在源码中存在，编译后丢弃（如 `@Override`）。
      - `CLASS`：编译后保存在 `.class` 文件中，但运行时 JVM 不加载（默认值）。
      - `RUNTIME`：不仅编译进 `.class`，还会在运行时加载到 JVM，可通过反射获取。

👉 示例：

```java
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnno {}
```

表示 `@MyAnno` 在运行时仍然可见，可以通过反射获取。

------

## 3. 总结对比表

| 元注解       | 属性名 | 类型            | 可选值                                                       | 作用                   |
| ------------ | ------ | --------------- | ------------------------------------------------------------ | ---------------------- |
| `@Target`    | value  | ElementType[]   | TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE, ANNOTATION_TYPE, PACKAGE, TYPE_PARAMETER, TYPE_USE, MODULE | 限定注解能用在什么位置 |
| `@Retention` | value  | RetentionPolicy | SOURCE, CLASS, RUNTIME                                       | 限定注解的生命周期     |
