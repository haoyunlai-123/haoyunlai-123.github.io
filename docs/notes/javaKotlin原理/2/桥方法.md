”编译完成之后，inner 实例内部会有指向 outer 实例的引用this$0，但是简单的outer.name是无法访问 private 属性的。从反编译的结果可以看到，outer 中会有一个桥方法static String access$000(OutterClass)，恰好返回 String 类型，即 userName 属性。正是通过这个方法实现内部类访问外部类私有属性。
 “这段话举个例子

下面用一段最小可运行代码把这段话的全过程演示出来——**内部类凭什么能摸到外部类的 `private` 成员**，以及编译器在背后偷偷干的俩活：

1. 在内部类里藏一个 `外部类.this` 引用（字段名 `this$0`）。  
2. 给外部类生成一个**合成桥方法** `access$000()`，返回那个 `private` 值。

------------------------------------------------
源码（合法，可直接跑）

```java
public class Outer {
    private String name = "Outer-private";

    public class Inner {
        public void printOuterName() {
            // 看起来好像直接用了 private 属性
            System.out.println("Inner sees: " + name);
        }
    }

    public static void main(String[] args) {
        Outer outer = new Outer();
        Inner inner = outer.new Inner();
        inner.printOuterName();
    }
}
```

一句话总结  
内部类里写 `name` 只是**语法糖**，编译后实际变成：

```java
System.out.println("Inner sees: " + Outer.access$000(this$0));
```

正是靠 `access$000` 这个**合成桥方法**，内部类才能“合法”拿到外部类的 `private` 成员。

桥方法反编译展示出来我看看

好，把刚才那个**合成桥方法**直接用你能敲的 Java 代码写出来，就是：

```java
// 编译器偷偷塞进来的包可见静态方法
static String access$000(Outer outer) {
    return outer.userName;   // 直接返回外部对象的私有字段
}
```

特征  
- 名字固定格式：`access$ + 序号`（因此**不要**自己再取名 `access$000`，会被 javac 拒绝）。  
- 无 `public` 修饰，仅包可见，防止被你意外调用。  
- 接收一个外部类实例参数，内部走 `return 私有字段;`——这就是内部类能“看”到 `private userName` 的真正入口。
